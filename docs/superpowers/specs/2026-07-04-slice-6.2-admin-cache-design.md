# Slice 6.2 — admin cache + flags (design)

**Goal:** SourceMod's admin authorization — a host-global admin cache (`SteamID64 → flags`) loaded from
an `admins.json` file *and* mutable at runtime, the `ADMFLAG_*` flag model, and admin-gated command
registration. Retrofit `sm_say` to require `ADMFLAG.CHAT`. This is the real "permissions" for SM parity
(distinct from the plugin-capability idea dropped in 5E — SM's "permissions" are admin command flags).

**Status:** design approved. JSON admins file + named SM flags + a **host-global two-tier cache** (file +
runtime); command gating via the **SM trio** `Commands.register`/`registerServer`/`registerAdmin` (option
2's direct flags), `registerAdmin` enforced through an admin-check hook that `@s2script/admin` installs.

**Branch base:** `main` (… + 6.1c merged).
**Cadence:** subagent-driven, merge-to-main-locally, Docker CS2 live gate.

Second rung of Slice 6 (base-plugin suite). Sequence: 6.1 command spine → 6.1b/c chat → **6.2 admin
(this)** → 6.3 target-resolution + kick + slap.

## 1. The flag model (`ADMFLAG`, SM-parity bit values)

Named flags, exposed in TS as an `ADMFLAG` constant object; each maps to SourceMod's bit value:

| name | bit | SM letter | name | bit | SM letter |
|---|---|---|---|---|---|
| RESERVATION | 1<<0 | a | CONFIG | 1<<8 | i |
| GENERIC | 1<<1 | b | CHAT | 1<<9 | j |
| KICK | 1<<2 | c | VOTE | 1<<10 | k |
| BAN | 1<<3 | d | PASSWORD | 1<<11 | l |
| UNBAN | 1<<4 | e | RCON | 1<<12 | m |
| SLAY | 1<<5 | f | CHEATS | 1<<13 | n |
| CHANGEMAP | 1<<6 | g | ROOT | 1<<14 | z |
| CONVARS | 1<<7 | h |  |  |  |

**`ROOT` grants everything** — a `hasFlags(required)` check passes if the admin has `ROOT` OR
`(flags & required) === required`. Flag sets are a `u64` bitmask (15 bits ≪ 2⁵³, so a plain JS number is
exact; no bigint needed).

## 2. The host-global cache (core)

The admin list must be shared across all plugins (plugin A's runtime `Admin.add` must be visible to plugin
B's gating), and V8 contexts are isolated — so the cache lives in **core**, not per-context JS. Two tiers
(SM parity — a persistent file source + a runtime source):

- `ADMIN_FILE:    RefCell<HashMap<String /*steamid64*/, u64 /*mask*/>>` — the file tier.
- `ADMIN_RUNTIME: RefCell<HashMap<String, u64>>` — the runtime tier (`Admin.add`).

Lookup unions both (`runtime_bits | file_bits`). Natives (engine-generic; a SteamID + a bitmask are not
game facts):

```
__s2_admin_set(steamid: string, flags: number, runtime: bool)   // set/overwrite in a tier
__s2_admin_get(steamid: string) -> number                       // union of both tiers; 0 = not an admin
__s2_admin_remove(steamid: string, runtime: bool)               // remove from a tier
__s2_admin_clear_file()                                          // wipe the file tier (for reload)
```

A `client_steamid(slot) -> string` op (ABI-appended after `client_print`) via
`IVEngineServer2::GetClientXUID(CPlayerSlot(slot))` returns the caller's SteamID64 as a decimal string
(`"0"`/empty for bots, unauthenticated, or invalid slots — never an admin; `slot < 0` = server console).

## 3. `@s2script/admin` (engine-generic — the typed API over the core cache)

```ts
export const ADMFLAG: { RESERVATION: number; …; CHAT: number; …; ROOT: number };

export interface AdminInfo { readonly steamId: string; readonly flags: number; hasFlags(required: number): boolean; }
export declare const Admin: {
  add(steamId: string, flags: number): void;      // runtime tier (not persisted) — SM AddUserToCache
  remove(steamId: string): void;                  // runtime tier
  get(steamId: string): AdminInfo | null;         // union of tiers; null if not an admin
  forSlot(slot: number): AdminInfo | null;         // resolve slot -> SteamID (client_steamid) -> get
  reload(): void;                                  // re-read admins.json into the file tier
};
```

- `flags` is a bitmask OR of `ADMFLAG.*` (e.g. `ADMFLAG.KICK | ADMFLAG.CHAT`); `Admin.add` accepts the
  same. `hasFlags(required)` is `(flags & ADMFLAG.ROOT) !== 0 || (flags & required) === required`.
- **File load (once):** on the FIRST plugin that imports `@s2script/admin`, the module reads
  `addons/s2script/configs/admins.json` (via the 5E.2 `config_read` op — reused), parses it, and
  `__s2_admin_set(steamid, mask, /*runtime=*/false)` for each entry; a core one-shot guard
  (`__s2_admin_mark_loaded() -> bool`, returns the prior state) ensures it runs once (subsequent imports
  skip). `Admin.reload()` clears the guard + `__s2_admin_clear_file()` + re-parses. **Auto-generate:** if
  the file is absent, write an empty commented template via `config_write` (like config).
- **File format** — `addons/s2script/configs/admins.json`:
  ```jsonc
  { "76561199000000001": ["root"],
    "76561199000000002": ["kick", "ban", "slay", "chat"] }
  ```
  string flag NAMES (case-insensitive), mapped to bits by the module (unknown name → WARN + skip).
- **The check hook:** the module installs `globalThis.__s2_admin_check = (slot, requiredMask) => { const a =
  Admin.forSlot(slot); return a ? a.hasFlags(requiredMask) : false; }` so `@s2script/commands` can enforce
  admin flags without depending on `@s2script/admin` (§4). `slot < 0` is handled by commands (root) before
  the hook is consulted.

Also adds `player.steamId` (string) to the CS2 `Player` (via `client_steamid` on the player's slot) — a
nice-to-have alongside `player.userId`.

## 4. The command trio (`@s2script/commands` — SM RegConsole/Server/AdminCmd)

```ts
Commands.register(name: string, handler: (ctx: CommandContext) => void): void;        // console — anyone (6.1, unchanged) [RegConsoleCmd]
Commands.registerServer(name: string, handler: (ctx: CommandContext) => void): void;  // server console only (callerSlot < 0) [RegServerCmd]
Commands.registerAdmin(name: string, flags: number, handler: (ctx: CommandContext) => void): void;  // admin-gated [RegAdminCmd]
```

Dispatch semantics (in the JS module, per-command closure — reusing 6.1's `ctx`):
- **`register`** — invoke always (any caller).
- **`registerServer`** — `callerSlot < 0` → invoke; else `ctx.reply("[SM] This command can only be run from the server console.")` + skip.
- **`registerAdmin(name, flags, handler)`** — `callerSlot < 0` → invoke (server/rcon = root); else consult
  `globalThis.__s2_admin_check(callerSlot, flags)`: `true` → invoke; `false` → `ctx.reply("[SM] You do not
  have access to this command.")` + skip. If the hook is **absent** (`@s2script/admin` not loaded in this
  context) → degrade: deny non-server callers + a warn-once (`registerAdmin used without @s2script/admin`).

`@s2script/commands` stays admin-agnostic (it calls a global hook it never defines); `@s2script/admin`
depends on `@s2script/commands`? No — neither imports the other. The coupling is the loosely-typed global
hook, installed by whichever plugin loads `@s2script/admin`.

## 5. Retrofit `sm_say` + the demo

`@s2script/basecommands` imports `@s2script/commands` + `@s2script/chat` + `@s2script/admin`, and
registers:

```ts
import { Commands } from "@s2script/commands";
import { Chat } from "@s2script/chat";
import { ADMFLAG } from "@s2script/admin";

export function onLoad(): void {
  Commands.registerAdmin("sm_say", ADMFLAG.CHAT, (ctx) => {
    const msg = ctx.argString.trim();
    if (!msg) { ctx.reply("Usage: sm_say <message>"); return; }
    Chat.toAll("[SM] " + msg);
  });
}
```

Server console / rcon (root) runs it; a non-admin player is denied.

## 6. Degrade-never-crash

Absent `admins.json` → auto-generate + an empty cache (no admins). Malformed file / unknown flag name →
per-entry WARN + skip, no crash. `client_steamid` with no ops / a bot / an invalid slot → `"0"` → not an
admin. `registerAdmin` with no admin module → deny non-server + warn-once (fail-safe: never accidentally
grant). The core cache natives no-op degrade without ops. No path throws into plugin JS.

## 7. Testing & live gate

- **In-isolate (core):** the two-tier cache — `admin_set`/`get` (union file+runtime), `remove`,
  `clear_file`; `client_steamid` degrade; the one-shot load guard.
- **In-isolate (JS via `eval_in_context`):** `ADMFLAG` bit values; `Admin.add`/`get`/`hasFlags` (root ⇒ all;
  exact-subset match); `Admin.forSlot` (via a stubbed steamid); the file parse (a fixture JSON string →
  the right cache entries; unknown flag → skip); the trio dispatch — `register` always; `registerServer`
  console-only; `registerAdmin` (callerSlot<0 → allow; hook true → allow; hook false → deny + the reply;
  no hook → deny non-server).
- **In-isolate (CLI/vm):** the `@s2script/admin` + updated `@s2script/commands` `.d.ts` compile; the
  `basecommands` plugin passes the 5E.1 typecheck gate.
- **One sniper rebuild** (the `client_steamid` op + the admin-cache natives).
- **Live gate (de_inferno, rcon):** on first load the host auto-generates `admins.json`; the module loads
  it; `scripts/rcon.py "sm_say hi"` (server console = root) → `sm_say` runs (`[SM] hi`). A boot diagnostic
  logs `Admin.forSlot` for each connected bot slot (`steamId="0" admin=no`), proving the SteamID read +
  cache lookup. Seed a real SteamID64 in `admins.json` → `Admin.get(steamid)` resolves its flags (logged).
  **Caveat:** a *real non-admin player* visually seeing the denial can't be triggered on the bots-only
  gate (no human client, and bots can't run commands) — but the gating LOGIC is fully in-isolate proven,
  and the server-console=root path + the SteamID read + the cache load are live-verified.
