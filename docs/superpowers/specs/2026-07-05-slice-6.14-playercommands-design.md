# Slice 6.14 — `playercommands`: sm_slap + sm_slay + sm_rename

**Status:** design (brainstormed + approved 2026-07-05, feasibility-spiked)

**Goal:** Ship the SourceMod `playercommands` base plugin — `sm_slap`, `sm_slay`, `sm_rename` — as `@s2script/playercommands`, correcting the `sm_slap` misfiling (it currently lives in `basecommands`) and adding the two missing player-action staples. Continues the Slice-6 north star: the SM default-plugin suite, each an idiomatic separate plugin.

**Architecture:** One trivial move (`sm_slap`), one RE spike (`sm_slay` = a signature-resolved `CBasePlayerPawn::CommitSuicide` direct call), and one new engine-generic primitive (`sm_rename` = an entity string-write op + a netvar write). Both new actions need one sniper rebuild (a gamedata signature + two ABI ops). Core stays engine-generic; the CS2-specific facts (the CommitSuicide signature, the `m_iszPlayerName` offset) live in gamedata / the game layer.

**Non-goals / carried decisions:** `sm_slay`'s kill goes through the game's own death path (a raw `health=0` write is rejected — it produces a standing 0-HP zombie, never running `Event_Killed`). `sm_rename`'s client-visible propagation is the one genuinely uncertain outcome (CS2 caches names on some surfaces) — the netvar write + `notifyStateChanged` is built and **verified live**, with an `Events.fire("player_changename")` best-effort backstop and a documented caveat if a surface doesn't update. This slice does NOT build the `DispatchTraceAttack`-lethal-call fallback unless the CommitSuicide signature proves unresolvable/fragile.

---

## Evidence base (from the feasibility spike)

- **Damage/kill playbook already proven:** `DispatchTraceAttack` is sig-scanned + directly callable (`gamedata/core.gamedata.jsonc:62`, the install-time self-test `shim/src/s2script_mm.cpp:~1314` `reinterpret_cast<...>(addr)(...)`, trampoline `g_origDTA`). The sig-scan machinery (`shim/src/sigscan.*`, `LoadSignatures`, the 6.9 `.signatures` validation gate) is in place.
- **CommitSuicide string anchors located** on the pinned `libserver.so`: `"CommitSuicide"`@~0x8a0a28 and `"…committed suicide with world"`@~0x8e0c98 (from the damage-hooks memory). ABI: `CBasePlayerPawn::CommitSuicide(void* this, bool bExplode, bool bForce)`.
- **A prior failed attempt to salvage from:** branch `slice-6.8-slay` (`a085d5a`, unmerged) called CommitSuicide via ModSharp's **hardcoded vtable index 400** — the borrowed constant the RE doctrine forbids; it hit a valid-but-wrong function and silently no-op'd. Its **serial-gate + `.text`-range guard scaffolding** (`shim/src/s2script_mm.cpp:698-711` on that branch) is sound and reused; only the vtable call is replaced by a sig-resolved direct call.
- **Rename target (hard):** `m_iszPlayerName` @**2036** on `CBasePlayerController`, `char[128]` **inline** (the source of the read-only `player.playerName` getter, `games/cs2/js/schema.generated.js:~192`). No game function is used by ModSharp/CSSharp for rename → it is a netvar write, not a function call.
- **String WRITE is missing:** `__s2_ent_ref_read_string` + the `read_string` helper exist (`core/src/entity.rs:~64`, `core/src/v8host.rs:~481`); there is **no** `write_string` op/native, and codegen marks `char[N]` `writable:false`. So rename needs one new primitive.

## Architecture — three units

### Unit 1 — Plugin skeleton + `sm_slap` move (no RE, no rebuild)

Create `plugins/playercommands/` (`package.json` = `@s2script/playercommands`, `src/plugin.ts`, `tsconfig.json` extending `../../tsconfig.base.json`). **Move `sm_slap` verbatim** from `plugins/basecommands/src/plugin.ts` into it — the existing implementation (reliable `pawn.health = max(1, hp-damage)` via the generated setter + best-effort `pawn.setVelocity` knockback), `ADMFLAG.SLAY`, `Player.target` resolution. `basecommands` drops `sm_slap` + prunes any now-unused import; its remaining commands (kick/map/cvar/who/rcon/exec/`sm`) are untouched. Mirrors the 6.13-B basechat extraction. Build the `.s2sp`; typecheck gate green for both plugins.

### Unit 2 — `sm_slay` via `CommitSuicide` direct-call-by-signature (the RE spike; sniper rebuild)

**RE spike (offline, on the pinned `libserver.so`, doctrine-compliant):**
1. From the located `"CommitSuicide"` / `"…committed suicide with world"` string anchors, follow the `lea rip+disp32` xref to the referencing function's start; confirm it is `CBasePlayerPawn::CommitSuicide` (cross-check against the `slice-6.8-slay` findings + the ModSharp/CSSharp gamedata, which name the same function — used only as HINTS, per the doctrine).
2. Extract a stable prologue byte pattern; verify **UNIQUE** in our `libserver.so` via the existing `sigscan` `CountPattern`; add a `CommitSuicide` entry to `gamedata/core.gamedata.jsonc` `.signatures` (`module: libserver.so`, `resolve: "direct"`). The 6.9 validation gate must print it OK at boot (a stale sig screams, never silently no-ops — the whole point vs the vtable-400 failure).

**Shim + op:**
- The shim resolves the signature (via the existing `LoadSignatures`) → stores the `CommitSuicide` fn pointer.
- A new **engine-generic ABI op `pawn_commit_suicide`** (appended after the last existing op — C header + Rust mirror + both test literals) that takes an entity (index+serial), resolves the pawn pointer **serial-gated** (reuse `s2_deref_handle`/the entity lookup + the salvaged `.text`-range guard), and calls `fn(pawn, /*bExplode=*/false, /*bForce=*/true)`. Degrade to no-op (never crash) if the sig is unresolved or the entity is stale. `CBasePlayerPawn` is a Source2 base type → the op is engine-generic (the CS2-ness is only the gamedata signature).
- A `__s2_pawn_commit_suicide(idx, serial)` native.

**CS2 API + command (game layer):**
- `pawn.slay()` (or `Player.slay()` — resolve the pawn via `player.pawn`, call the native). Serial-gated `T|null`-safe.
- `sm_slay <target>` (`ADMFLAG.SLAY`) in `@s2script/playercommands`: `Player.target` resolution (SM selectors), per matched player `p.pawn?.slay()`.

**Fallback (documented, NOT built unless needed):** if the prologue sig can't be made stable/unique, call the already-resolved `DispatchTraceAttack(victimPawn, &lethalInfo, 0, 0)` with a zero-init `CTakeDamageInfo` (`m_flDamage`@68 ≥ HP, a generic `m_bitsDamageType`@76) — no new RE, worse attribution (world-damage death).

### Unit 3 — `sm_rename` via an entity string-write primitive (new primitive; sniper rebuild)

**New engine-generic primitive (core + shim):**
- ABI op `ent_write_string` + native `__s2_ent_ref_write_string(idx, serial, off, maxLen, str)` — **mirror `read_string`**: one serial-gated lookup, then a **bounded, NUL-terminated `memcpy`** of at most `maxLen-1` bytes into the inline `char[maxLen]` at `off` (truncate-safe; the raw pointer never crosses to JS). A pure `core/src/entity.rs::write_string(ptr, off, maxLen, bytes)` helper (V8-free, unit-tested: writes + NUL-terminates + truncates at `maxLen-1`). Degrade to no-op on stale serial. Engine-generic (a string at a schema offset — no game type).
- `EntityRef.writeString(off, maxLen, str)` in `@s2script/entity`.

**CS2 API + command (game layer):**
- `player.setName(name)` — writes `m_iszPlayerName` (offset **live-resolved** via `__s2_schema_offset`, never baked) as `char[128]` via `writeString`, then `notifyStateChanged(off)` to mark it dirty for re-replication. Hand-written on the `Player` prototype (one field — simpler than enabling the codegen `str` setter; the codegen `char[N]`-writable path stays deferred).
- `sm_rename <target> <newname>` (`ADMFLAG.SLAY`) in `@s2script/playercommands`: `Player.target` (single match — a rename is destructive/1-target like kick; reject ambiguous multi), sanitize/trim the new name (bound to 127 bytes, strip control bytes), `p.setName(newname)`.
- **Propagation backstop:** after the write, best-effort `Events.fire("player_changename", { userid, oldname, newname })` (5D.3 `Events.fire` exists) so any client surface that listens for the rename event updates. If the live gate shows a surface (scoreboard/kill-feed) not updating from the netvar write, document the caveat — there is no clean game-function fallback.

## Data flow

- `sm_slay <t>` → `Player.target` → `p.pawn.slay()` → `__s2_pawn_commit_suicide(idx,serial)` → op → shim serial-gates the pawn + calls the sig-resolved `CommitSuicide(pawn,false,true)` → CS2 runs its real death path (ragdoll, death notice, respawn timer).
- `sm_rename <t> <name>` → `Player.target` (1) → `p.setName(name)` → `writeString(m_iszPlayerName off, 128, name)` + `notifyStateChanged` + `Events.fire("player_changename")`.

## Error handling / degrade

- Sig unresolved (bad/stale signature) → `pawn_commit_suicide` no-ops + the 6.9 gate flags it at boot; `sm_slay` replies "slay unavailable" rather than a silent nothing.
- Stale/`null` pawn (target left) → serial-gate → no-op, no crash.
- Bot targets are fine for slay (a direct pawn-fn call, no netchannel needed — unlike chat/print).
- `sm_rename` empty/oversized name → trimmed/bounded; multi-target → rejected (single-target only).
- `notifyStateChanged` / `Events.fire` failures degrade to no-op.

## Testing

- **Unit (core, node/cargo):** `entity.rs::write_string` — writes, NUL-terminates, truncates at `maxLen-1`, leaves nothing past the bound; `pawn_commit_suicide` / `ent_write_string` natives-degrade-without-ops asserts (mirror the existing damage-op degrade tests).
- **Boundary gates:** `check-core-boundary.sh` + `test-boundary-nameleak.sh` green (no CS2 names in `core/src`; `CommitSuicide` sig + `m_iszPlayerName` stay in gamedata/game layer).
- **Typecheck gate:** `plugins/playercommands` + `plugins/basecommands` typecheck full-strict.
- **Sig validation:** the 6.9 gate prints `CommitSuicide` resolved-OK at boot.
- **Live gate (de_inferno/de_dust2, `bot_quota 2`, + the user as a client):**
  - `sm_slap <bot> N` still works from its new home; `basecommands` no longer has `sm_slap`.
  - `sm_slay <bot>` → the bot **dies** (ragdoll + death notice), server keeps ticking, `RestartCount=0` (bots suffice — a direct call).
  - `sm_rename <you> NewName` → the user (as client) observes whether their **scoreboard name** changes; record which surfaces update (scoreboard / kill-feed / chat prefix). Document any non-updating surface.

## Out of scope (later)

`sm_rename` on the codegen `char[N]`-writable path (hand-written for one field here); teleport/velocity that survives physics; the `DispatchTraceAttack`-lethal fallback (only if the sig fails); `sm_slay`'s explode variant (`bExplode=true`); immunity checks; the remaining SM default plugins.
