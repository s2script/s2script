# Slice 6.7 — `sm_cvar` (cvar get/set) design

**Goal:** The canonical SM `sm_cvar <name> [value]` — read a cvar's value, or set it. Adds a **tier1-free
cvar-read primitive** to `@s2script/server` (`Server.getCvar`), reusing `Server.command` for the set.

**Status:** design (spike complete). GET reads `ConVarData` directly (no tier1 method calls); SET via the
console. Live-verifiable with a known cvar.

**Branch:** `slice-6.7-cvar` (off main `f9c149f`).

---

## 1. The cvar-read primitive (the spike result)

The clean SDK accessors are **non-inline** (`ConVarData::ValueOrDefault`, `ConVarRefAbstract::GetString` →
`CUtlString`) → they'd reintroduce the tier1/`dlopen` cascade (5D.1). The **tier1-free** path (shim,
`cvar_get(name) -> const char*`):

1. `ConVarRef ref = s_pCvar->FindConVar(name, false);` — a vtable call on the held `ICvar` (safe).
2. `if (!ref.IsValidRef()) return "";` — inline validity check.
3. `ConVarData* data = s_pCvar->GetConVarData(ref);` — vtable call (safe); null-guard.
4. `EConVarType t = data->GetType();` — inline field read (`m_eVarType`).
5. Read the current value directly from `m_Values` (the LAST field of `ConVarData`, so its offset =
   `sizeof(ConVarData) - sizeof(CVValue_t) * MAX_SPLITSCREEN_CLIENTS`; `m_Values` is `CVValue_t`-aligned and
   trailing → no padding after it):
   ```cpp
   const size_t VOFF = sizeof(ConVarData) - sizeof(CVValue_t) * MAX_SPLITSCREEN_CLIENTS;
   CVValue_t* v = reinterpret_cast<CVValue_t*>(reinterpret_cast<char*>(data) + VOFF);
   ```
   Then `snprintf` into a static buffer by type (all reads are public union members / inline):
   - `Bool`→`v->m_bValue?"1":"0"`, `Int16/Int32`→`%d`, `UInt32`→`%u`, `Int64`→`%lld`, `UInt64`→`%llu`,
     `Float32`→`%g v->m_fl32Value`, `Float64`→`%g`.
   - `String` → `v->m_StringValue.Get()` (`CUtlString::Get` is inline). GUARDED by `t==String` (never read
     the CUtlString on non-string data).
   - `Color`/`Vector*`/`Qangle`/`Invalid` → `"<type>"` (deferred; rare for `sm_cvar`).

**Risk:** the read assumes the pinned SDK `ConVarData`/`CVValue_t` layout matches the game ABI (the SDK can
lag Valve — a charter concern). Mitigation: the live gate reads a KNOWN cvar (`sv_gravity` default 800) and
confirms the value — a wrong layout shows as garbage, caught immediately. No new gamedata offset (the
layout comes from the pinned header).

## 2. `@s2script/server` additions (engine-generic)

```ts
Server.getCvar(name: string): string;          // "" if the cvar doesn't exist / unsupported type
Server.setCvar(name: string, value: string): void;   // = Server.command(name + " " + value)
```
`getCvar` → the `cvar_get` op (ABI-appended after `damage_victim`) → `__s2_cvar_get` native. `setCvar` is a
thin `Server.command` wrapper (no new op; the console sets any type correctly). Injection: `setCvar`
sanitizes nothing itself, but `sm_cvar` validates the name.

## 3. `sm_cvar <name> [value]` (`ADMFLAG.CVARS`) — `@s2script/basecommands`

```ts
Commands.registerAdmin("sm_cvar", ADMFLAG.CVARS, (ctx) => {
  const parts = ctx.argString.trim().split(/\s+/);
  const name = parts[0] || "";
  if (!name || !/^[A-Za-z0-9_]+$/.test(name)) { ctx.reply("Usage: sm_cvar <name> [value]"); return; }
  if (parts.length < 2) { ctx.reply("[SM] " + name + " = " + Server.getCvar(name)); return; }  // GET
  const value = parts.slice(1).join(" ");
  Server.setCvar(name, value);                                                                  // SET
  ctx.reply("[SM] " + name + " = " + Server.getCvar(name));   // read back to confirm
});
```
Name sanitized (`^[A-Za-z0-9_]+$`) since `setCvar` builds a console command (`;`-injection guard). Value
may contain spaces (a string cvar). GET of a nonexistent cvar → `""`.

## 4. Boundary & degrade

`cvar_get` is engine-generic (a cvar name → string; Source 2, not CS2) → the op + `Server.getCvar` live in
core/shim. `sm_cvar` is CS2 basecommands. Degrade: no `ICvar` / unknown cvar / null data → `""`; the native
→ `""` without the op; unsupported type → `"<type>"`. Nothing throws.

## 5. Testing & live gate

- **In-isolate (core):** `__s2_cvar_get('x')` → `""` without the op; `Server.getCvar` wired.
- **Examples typecheck:** `basecommands` compiles with `Server.getCvar/setCvar`.
- **One sniper rebuild.**
- **Live gate (rcon):** `sm_cvar sv_gravity` → `"800"` (the KNOWN default — confirms the layout read is
  correct, not garbage); `sm_cvar sv_gravity 400` → sets + reads back `"400"`; `sm_cvar mp_maxrounds`
  (int), a float cvar, and a string cvar (`hostname`) → plausible values; `sm_cvar nonexistent` → `""`;
  `RestartCount=0`.

## 6. Out of scope

Cvar flags/help/default in the reply; `Color`/`Vector`/`Qangle` cvar formatting (→ `"<type>"`); a protected/
cheat-cvar guard (SM's `sm_cvar` flag checks); a typed `Server.getCvarInt/Float` (string only for now);
change-callback subscription. Note as TODOs and stop.
