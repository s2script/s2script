# Slice 6.6 — `OnTakeDamage` pre-hooks — feasibility spike findings

**Goal of the slice:** SDKHooks-equivalent damage pre-hooks — `Damage.onPre(handler)` where the handler
reads the damage event `{victim, attacker, inflictor, damage, damageType, ...}` and may **block** (zero
damage / suppress) or **modify** (change `m_flDamage`, etc.) via a `HookResult`. Makes `sm_slap` "real
damage", and unblocks the entire class of damage-modifying/blocking plugins.

**This document = the spike (feasibility only, no build yet).** Every unknown from the pitch is now
resolved except two, and the one open *decision* is the detour mechanism.

---

## What the spike established (all GREEN)

1. **The damage struct is schema-registered → no struct RE needed.** `CTakeDamageInfo` is in the committed
   `schema-catalog.json` with every field an author needs, resolvable **live** via the existing
   `__s2_schema_offset` accessor:
   - `m_flDamage` @68 (float, read+**write** for modify), `m_flTotalledDamage` @72
   - `m_hAttacker` @60, `m_hInflictor` @56, `m_hAbility` @64 (CEntityHandle → decode to `EntityRef` via
     the existing `__s2_handle_decode`)
   - `m_bitsDamageType` @76 (int), `m_iDamageCustom` @80, `m_iAmmoType` @84
   - `m_vecDamageForce`/`m_vecDamagePosition`/`m_vecDamageDirection` (vectors)
   So reading/modifying the damage info reuses the whole 5B/5C schema stack — **zero new struct plumbing.**

2. **The damage function is findable by a unique signature — the 5D.2 sigscan infra already handles it.**
   The CS2 damage entry is `CBaseEntity::DispatchTraceAttack` (its body holds the `"CBaseEntity::
   TakeDamageOld: damagetype %d…"` debug strings). Linux prologue signature (from the ModSharp CS2
   framework's `server.games.jsonc`, cross-referenced):
   ```
   55 48 89 E5 41 57 41 56 41 55 49 89 FD 41 54 49 89 F4 53 48 89 D3 48 83 EC ? 48 85 D2
   ```
   **Validated against the pinned `libserver.so`** (39.2 MB): **UNIQUE — exactly 1 match at `0x15ce2e0`**.
   Resolves cleanly with the existing `FindPattern` + `.signatures` gamedata section + `LoadSignatures`
   (5D.2). It's a per-update gamedata signature (the treadmill — offsets/patterns move per patch).

3. **Entity refs fall out of what we have.** SysV ABI at the detour: `rdi = this` (the **victim** entity →
   its `CEntityIdentity` → `EntityRef` via the entity system), `rsi = &CTakeDamageInfo` (the info ptr, kept
   host-side, exposed only through a block-scoped schema accessor). `m_hAttacker`/`m_hInflictor` → decode
   to `EntityRef`. No raw pointer crosses to JS.

4. **The JS API + block/modify mirror 5D.1/5D.3 exactly.** A core `damage_mux` (snapshot + `run_chain`
   `HookResult` collapse + `remove_by_owner`, like `event_mux`), a block-scoped `DamageInfo` accessor
   (synchronous-only, like `GameEvent` — read via schema, `setDamage()` writes `m_flDamage`), `HookResult`
   → `Continue`/`Changed` (apply the writes, call original) / `Handled`/`Stop` (**block**: skip the
   original or zero the damage). Re-entrancy via `try_borrow_mut` (the isolate-borrow rule).

## The one new piece of infrastructure — the function detour

`DispatchTraceAttack` is an **internal member function found by signature**, NOT a registered interface
(unlike 5D.3's `IGameEventManager2::FireEvent`, which was a vtable `SH_ADD_HOOK`). So it needs a **function
detour** (overwrite the prologue with a jump to our handler + a trampoline that relocates the stolen
prologue bytes to call the original). The shim today has SourceHook (interface/vtable hooks) + the sigscan
(global-pointer resolution) but **no function-detour mechanism** — this is the central new capability.

A robust prologue detour needs x86-64 instruction-length decoding (to relocate whole instructions into the
trampoline). Options considered:
- **Vendor MinHook** — small, self-contained, MIT, x64, *built-in* length disassembler (HDE64). Static
  `.a`, so NO tier1/`dlopen` cascade (the 5D.1 concern). The standard, minimal choice. **← recommended.**
- **Vendor safetyhook** — modern, more robust, but bundles Zydis (a heavier vendor).
- **Hand-roll a minimal trampoline** — no new dep, but I must hand-write x64 prologue relocation (fragile,
  reinvents a length disassembler). Not worth it.
- **SourceHook manual vtable hook** — zero new dep, BUT only works if `DispatchTraceAttack` is virtual at a
  known vtable index. ModSharp resolves it as a *function address* (detour), which suggests it is not
  hooked as a vtable slot; unconfirmed whether it's virtual. If it turns out virtual, this is cleanest.

## The one open risk — live verification on the bots-only gate

`DispatchTraceAttack` fires on trace/bullet damage. On the 2-bot gate, bots on opposite teams DO shoot each
other, but **timing is nondeterministic** (the `live-gate-boot-window` note: MAXPLAYERS=2 combat is
unreliable *for timing*, not impossible). Mitigation: the first deliverable is a **read-only logging hook**
("does the hook fire") — verified by waiting for any bot hit; then block/modify. A scripted deterministic
trigger (e.g. `sv_cheats` + an ignite/hurt path) is a fallback to investigate. This risk is why the slice
is staged read-first.

## Proposed staging (thin vertical thread first)

1. **The detour engine + sigscan wiring + a read-only logging hook.** Vendor the hook lib; add the
   `DispatchTraceAttack` `.signatures` entry; detour it; log `{victim, attacker, damage}` from the core.
   Live-gate: prove the hook fires on a bot hit. (The riskiest thread first — proves detour + sig + live.)
2. **The core `damage_mux` + `Damage.onPre` JS API + the block-scoped `DamageInfo` accessor (read).**
3. **Block + modify** (`HookResult` collapse → suppress / write `m_flDamage`) + the typed `.d.ts`.
4. Live gate: a hooked victim takes reduced/zero damage; docs.

## Boundary

`DispatchTraceAttack`/`CTakeDamageInfo`/the detour are **Source 2 engine-generic** (not CS2-specific — a
`CTakeDamageInfo` exists on any Source 2 game), so the mux + API live in **core** + the shim owns the
detour/sigscan; the field names are resolved via schema (data, not code). `@s2script/damage` is an
engine-generic module. Both boundary gates stay green.
