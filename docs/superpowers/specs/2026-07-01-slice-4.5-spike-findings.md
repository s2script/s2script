## Step 1: JSON String Carrier Across Two Contexts

**GO.**

### Confirmed rusty_v8 sequence (stringify side, in context A)

```rust
// 1. Open isolate scope and enter the producer's context.
let mut hs_storage = v8::HandleScope::new(&mut host.isolate);
let mut hs = unsafe { std::pin::Pin::new_unchecked(&mut hs_storage) }.init();
let hs = &mut hs;
let ctx_local = v8::Local::new(hs, &g_ctx);      // materialise Global<Context>
let scope = &mut v8::ContextScope::new(hs, ctx_local);

// 2. Build the value to serialise (any JS expression).
let src = v8::String::new(scope, "({n:41,s:'hi',arr:[1,2,3]})").unwrap();
let val = v8::Script::compile(scope, src, None).unwrap().run(scope).unwrap();

// 3. Fetch JSON.stringify via the global chain.
let global   = scope.get_current_context().global(scope);
let jk       = v8::String::new(scope, "JSON").unwrap();
let json_obj = v8::Local::<v8::Object>::try_from(global.get(scope, jk.into()).unwrap()).unwrap();
let sk       = v8::String::new(scope, "stringify").unwrap();
let stringify = v8::Local::<v8::Function>::try_from(json_obj.get(scope, sk.into()).unwrap()).unwrap();

// 4. Call: Some(result) on success, None if stringify throws.
let recv: v8::Local<v8::Value> = json_obj.into();
let json_str: String = stringify.call(scope, recv, &[val])
    .unwrap()
    .to_rust_string_lossy(scope);
// json_str is a plain Rust String — context-free, no V8 handles.
```

### Parse side (in context B)

The plain Rust `String` is passed into any other context's scope directly.
No handle transfer, no GC entanglement:

```rust
// json_str: plain Rust String, no context affinity.
eval_in_context("consB", &format!(
    "const _o = JSON.parse('{}'); globalThis.__n = _o.n;",
    json_str          // JSON uses double-quotes; no single-quote escaping needed
)).unwrap();
```

**Assertion confirmed:** `read_i32_in("consB", "__n") == 41` and
`read_string_in("consB", "__s") == "hi"`.

## Step 2: Non-Serialisable Values

**GO.**

### Cyclic object — `Function::call` returns `None`

A cyclic object (`var a = {}; a.a = a`) causes `JSON.stringify` to throw a
`TypeError`. In rusty_v8, a throwing `Function::call` returns `None`.
A `TryCatch` must be active to absorb the exception; without it the isolate
retains a pending exception that can corrupt later calls.

```rust
// TryCatch absorbs the TypeError so the isolate stays clean.
let mut tc_storage = v8::TryCatch::new(scope);
let mut tc = unsafe { std::pin::Pin::new_unchecked(&mut tc_storage) }.init();
let tc = &mut tc;

let recv: v8::Local<v8::Value> = json_obj.into();
let result = stringify.call(tc, recv, &[cyclic]);
assert!(result.is_none());   // None ↔ thrown
```

**Confirmed:** `result.is_none() == true`.

### JSON loss-of-type semantics (spec, not separately tested)

Standard JSON semantics apply: a `function` or `undefined` value at
the top level stringifies to `undefined` (not a string, `stringify`
returns the JS value `undefined` — not None in rusty_v8, but the
resulting Rust string will be `"undefined"`). Inside an array it
becomes `null`; inside an object it is omitted. Since `iface_call`
always wraps args in an array (`[arg1, arg2, ...]`), a function
argument becomes `null` — lossy but **does not throw**. Only
structurally-unrepresentable values (cyclic references, `BigInt`)
cause `Function::call` to return `None`, which maps to
`InterfaceValueNotSerializable`.

## Step 3: ES Proxy Trap → Native → Cross-Context `Global<Function>` Call

**GO.**

### Capturing a producer function as `Global<Function>`

```rust
// In prodA's context scope:
let src  = v8::String::new(scope, "(function greet(x){ return x * 2; })").unwrap();
let func_val = v8::Script::compile(scope, src, None).unwrap().run(scope).unwrap();
let func = v8::Local::<v8::Function>::try_from(func_val).unwrap();
// scope.as_ref() yields &Isolate (AsRef<Isolate> for every scope type).
let g_fn: v8::Global<v8::Function> = v8::Global::new(scope.as_ref(), func);
// g_fn is context-free; can be stored in a thread-local or struct field.
```

### Entering a different context to call the Global

Inside a native callback (`scope: &mut v8::PinScope`), a nested
`ContextScope` switches the active context without requiring a new
`HandleScope`. Values created inside the inner scope are stored in the
outer handle scope's handle area and remain valid after the
`ContextScope` drops. To avoid cross-scope `Local` lifetime issues,
extract primitive results (e.g., `f64`) before the `ContextScope` drops:

```rust
// scope is the native callback's PinScope (consB's context active).
let result: f64 = {
    let prod_ctx_local = v8::Local::new(scope, &g_ctx);       // materialise producer ctx
    let prod_scope = &mut v8::ContextScope::new(scope, prod_ctx_local); // enter prodA

    let mut tc_storage = v8::TryCatch::new(prod_scope);
    let mut tc = unsafe { std::pin::Pin::new_unchecked(&mut tc_storage) }.init();
    let tc = &mut tc;

    let prod_fn  = v8::Local::new(tc, &g_fn);                 // materialise Global<Function>
    let recv: v8::Local<v8::Value> = v8::undefined(tc).into();
    let x_val:  v8::Local<v8::Value> = v8::Number::new(tc, call_arg).into();

    prod_fn.call(tc, recv, &[x_val])
           .and_then(|v| v.number_value(tc))
           .unwrap_or(0.0)
    // tc, prod_scope drop here → consB context restored in V8
};
// Back in consB's context; result is a plain Rust f64.
rv.set(v8::Number::new(scope, result).into());
```

### The Proxy trap wiring (consumer side, pure JS)

```javascript
const proxy = new Proxy({}, {
    get(_, prop) { return (...args) => __s2_probe(prop, args); }
});
globalThis.__result = proxy.greet(7);   // → 14
```

`__s2_probe` is installed on consB's global object with `set_native`
after `create_plugin_context("consB")`.

**Assertion confirmed:** `proxy.greet(7) === 14` observed in consB
(`read_i32_in("consB", "__result") == 14`).

## Key Observations for Task Implementation

1. **`ContextScope` nesting inside callbacks works.** A native
   callback's `scope: &mut v8::PinScope` can host a nested
   `ContextScope` that switches to a different plugin's context and
   back, with no special handling required. V8 restores the prior
   context when the inner `ContextScope` drops.

2. **Extract primitives before the inner `ContextScope` drops.** Return
   `Local<Value>` Locals across a ContextScope boundary triggers Rust
   lifetime errors (the Local's lifetime is tied to the inner scope).
   The solution is to extract the value as a Rust primitive (`f64`,
   `i32`, `bool`) inside the inner block, then construct a new Local
   in the outer scope. For objects this means an additional
   `JSON.stringify` → Rust String → `JSON.parse` round-trip.

3. **The JSON carrier is a plain Rust `String`.** It crosses context
   boundaries as a Rust value — no V8 handle management required.
   The per-call overhead is two JS calls (stringify + parse) and one
   string allocation; acceptable for inter-plugin method calls.

4. **`Function::call` returns `None` for throwing JS.** Both cyclic
   JSON and any other throwing producer function map cleanly to the
   `iface_call` `InterfaceValueNotSerializable` / error path. A
   `TryCatch` must always be active to absorb the exception.

5. **`Global<Function>` + `Global<Context>` are the inter-plugin wire
   primitives.** Captured with `v8::Global::new(scope.as_ref(), local)`;
   materialised with `v8::Local::new(scope, &global)`. Both are
   context-free and can live in `HashMap`s or struct fields for the
   duration of the producer plugin's lifetime.

6. **`current_plugin(scope)` works correctly inside the probe native.**
   The callback is invoked in consB's context, so `current_plugin`
   returns `"consB"` — the correct consumer identity — even while the
   inner `ContextScope` (prodA) is active before it's entered.
