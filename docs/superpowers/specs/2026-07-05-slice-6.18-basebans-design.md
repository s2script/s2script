# Slice 6.18 — `basebans` (SteamID banning + connect-time enforcement)

**Goal:** Ship the SourceMod `basebans` plugin — `sm_ban` / `sm_unban` / `sm_addban` — with a persistent ban list that is *enforced at connect time* (a banned SteamID is rejected when it reconnects). `sm_banip` is deferred.

**Why now:** Highest-value remaining base plugin. Feasibility (2026-07-05 research) found a near-ideal path: the reject gate is `ISource2GameClients::ClientConnect`, a method on the *same interface we already SourceHook for `ClientCommand`*, and the persistence reuses the proven `admins.json` machinery one-for-one.

---

## Architecture

Three layers, all mirroring proven patterns:

1. **Connect enforcement (shim).** A second `SH_ADD_HOOK` on the already-acquired `m_gameClients` for `ClientConnect(CPlayerSlot, const char* name, uint64 xuid, const char* netid, bool, CBufferString* rejectReason) -> bool`. The SDK header (`hl2sdk/public/eiface.h:569-571`) documents: **returning `false` rejects the connection.** The hook reads the ban list (a shim→core call) keyed on `xuid` (the SteamID64, delivered as arg #3) and, if banned, `RETURN_META_VALUE(MRES_SUPERCEDE, false)` — mirroring the existing bool-returning `FireEvent` supercede at `s2script_mm.cpp:1640`. **MVP rejects with the generic engine message** (the `rejectReason` buffer is left untouched — writing it calls the tier1-imported `CBufferString::Insert`, the 5D.1 dlopen-cascade risk; a custom reject string is deferred).

2. **Ban store (core, host-global).** A `BAN_CACHE` thread_local `HashMap<String /*SteamID64*/, (i64 /*until_unix, 0=perm*/, String /*reason*/)>` beside the admin caches (`v8host.rs:353`). It **must** be host-global in core (not plugin-local JS) because its reader is the C++ connect hook, not a JS context — the identical constraint that forced the admin cache into core. Populated by JS via new `__s2_ban_*` natives; read by the shim via a new ffi export.

3. **Commands + persistence (JS).** An engine-generic **`@s2script/bans`** core prelude (embedded in `v8host.rs`, mirroring `@s2script/admin` at `v8host.rs:760-807`) exposes `Bans.add/remove/get/list/reload` and loads `bans.json` via the confirmed `__s2_config_read_raw("bans")` bridge. A CS2 **`@s2script/basebans`** plugin (`plugins/basebans/`) registers the three commands on the existing `Player.target` + `player.steamId` + `player.kick` + `registerAdmin` primitives.

### Data flow

```
sm_ban Alice 30 grief
  -> Player.target("Alice", callerSlot) -> [Player]  (NO_MULTI guard: refuse >1)
  -> sid = player.steamId
  -> Bans.add(sid, 30, "grief")   -> __s2_ban_set(sid, until, reason) -> BAN_CACHE
                                   -> rewrite bans.json (__s2_config_write_raw)
  -> player.kick("Banned: grief") (existing client_kick)

reconnect of a banned SteamID:
  engine -> ClientConnect(slot, name, xuid, ...) [shim hook]
         -> s2script_core_ban_check(xuid, time(NULL), reasonbuf, cap)  [ffi]
         -> reads BAN_CACHE, checks until==0 || until>now
         -> returns 1 (banned) -> hook logs + RETURN_META_VALUE(MRES_SUPERCEDE, false)  [rejected]
         -> returns 0 -> MRES_IGNORED, true  [connects normally]
```

## Components

### Core (`core/src/v8host.rs`, `core/src/ffi.rs`) — engine-generic

- `BAN_CACHE: thread_local RefCell<HashMap<String, (i64, String)>>` (declare beside `ADMIN_FILE`; clear in `shutdown`).
- Natives (mirror `__s2_admin_*`):
  - `__s2_ban_set(steamid: string, until: number, reason: string)` — insert/overwrite.
  - `__s2_ban_remove(steamid: string)` — remove.
  - `__s2_ban_get(steamid: string) -> string | null` — JSON `{until, reason}` or null (for `Bans.get`).
  - `__s2_ban_clear()` — wipe (for `Bans.reload` before re-parsing the file).
  - `__s2_ban_list() -> string` — JSON array `[{steamid, until, reason}]` (for `Bans.list` + rewrites).
  - `__s2_ban_mark_loaded() -> boolean` — one-shot load guard (mirrors `__s2_admin_mark_loaded`).
- ffi export `s2script_core_ban_check(xuid: u64, now: i64, out_reason: *mut c_char, cap: c_int) -> c_int` (mirror `s2script_core_dispatch_client_command`'s `catch_unwind`/panic->0 shape): look up `xuid.to_string()` in `BAN_CACHE`; banned iff present AND (`until == 0 || until > now`); if banned, bounded-copy `reason` into `out_reason` (for the shim's log line) and return `1`; else `0`. An expired entry reads as not-banned (optionally pruned lazily). Panic -> `0` (fail-open: never wedge connections on a core bug).
- `@s2script/bans` prelude (embedded JS, mirror `@s2script/admin`):
  - `__s2_ban_parseFile(text)` — JSON `{ "<steamid>": { "until": <unix|0>, "reason": "<str>" } }` -> `__s2_ban_set` each. Malformed -> skip-with-warn (degrade-never-crash).
  - `__s2_ban_load()` — `__s2_config_read_raw("bans")`; if null, `__s2_config_write_raw("bans", <valid-JSON _help template>)` (VALID JSON, per the admin-template live-gate fix — a `//`-comment template fails `JSON.parse`); then parse.
  - `__s2_ban_rewrite()` — `__s2_ban_list()` -> rebuild the `{steamid:{until,reason}}` object -> `__s2_config_write_raw("bans", ...)`.
  - `Bans.add(steamid, minutes, reason)` — `until = minutes>0 ? floor(Date.now()/1000)+minutes*60 : 0`; `__s2_ban_set` + `__s2_ban_rewrite`.
  - `Bans.remove(steamid)` — `__s2_ban_remove` + `__s2_ban_rewrite`. Returns boolean (was it present).
  - `Bans.get(steamid)` — `__s2_ban_get` parsed, or null.
  - `Bans.list()` — `__s2_ban_list()` parsed.
  - `Bans.reload()` — `__s2_ban_clear` + `__s2_ban_load`.
  - one-shot `if (!__s2_ban_mark_loaded()) __s2_ban_load();`
  - `globalThis.__s2pkg_bans = { Bans: __s2_bans };`

### Shim (`shim/src/s2script_mm.cpp`, `shim/include/s2script_core.h`) — engine-specific (Source2)

- Declare `s2script_core_ban_check` in `s2script_core.h`.
- `SH_DECL_HOOK6(ISource2GameClients, ClientConnect, SH_NOATTRIB, 0, bool, CPlayerSlot, const char*, uint64, const char*, bool, CBufferString*)`.
- `Hook_ClientConnect(slot, name, xuid, netid, unk1, rejectReason)`: log `slot/xuid/name/netid` (the diagnostic that also live-verifies the hook fires + `xuid` is the real SteamID); `int banned = s2script_core_ban_check(xuid, (int64)time(NULL), reasonbuf, sizeof(reasonbuf));` if `banned` -> log `"rejected banned player <name> (<reason>)"` + `RETURN_META_VALUE(MRES_SUPERCEDE, false)`; else `RETURN_META_VALUE(MRES_IGNORED, true)`.
- `SH_ADD_HOOK` beside the `ClientCommand` add (`s2script_mm.cpp:1152`); `SH_REMOVE_HOOK` beside its remove (`:1557`).
- **No new `S2EngineOps` op** — `ban_check` is a shim->core export, and `client_kick` already exists. ABI ops struct untouched.

### `@s2script/bans` types (`packages/bans/index.d.ts`) — engine-generic

`Bans.add(steamId, minutes, reason?) / remove(steamId) / get(steamId) / list() / reload()`; a `BanInfo = { until: number; reason: string }` value type. Mirrors `packages/admin/index.d.ts`.

### `@s2script/basebans` plugin (`plugins/basebans/`) — CS2

- `sm_ban <target> <minutes> [reason]` — `registerAdmin(ADMFLAG.BAN)`. Resolve `Player.target`; **NO_MULTI guard** (refuse a pattern matching >1, per the `sm_kick` destructive-command rule); read `player.steamId` (skip bots/steamid "0" with a reply); `Bans.add`; `player.kick("Banned: " + reason)`.
- `sm_unban <steamid> ` — `registerAdmin(ADMFLAG.UNBAN)`. `Bans.remove`; reply present/absent.
- `sm_addban <steamid> <minutes> [reason]` — `registerAdmin(ADMFLAG.BAN)`. Validate the steamid is digits; `Bans.add` (offline, no target resolution / no kick).
- `package.json` (`@s2script/basebans`, `s2script.apiVersion "1.x"`) + `tsconfig.json` (extends `../../tsconfig.base.json`).

## Testing

**In-isolate (cargo, hermetic — the provable-now surface):**
- `ban_check` returns 1 for a set-then-checked SteamID with `until=0` (perm) and with `until>now`; 0 for `until<now` (expired) and for an absent SteamID; fills `out_reason`.
- `__s2_ban_set` / `get` / `remove` / `list` / `clear` round-trip through `BAN_CACHE` (mirror the existing admin native tests at `v8host.rs:6083`).
- The `@s2script/bans` prelude parses a `{steamid:{until,reason}}` blob and populates via natives; malformed JSON degrades without throwing.

**Boundary gates:** core stays engine-generic (a SteamID/ban map is Source2-generic, exactly like the admin cache) — both `check-core-generic` gates green. `@s2script/basebans` is the only CS2 piece.

**Live gate (de_dust2 / de_inferno, the user as a real client — the honest ceiling):** `ClientConnect` is *not called for bots*, so the full reject path needs a human. Verify: (1) the connect log fires with the user's real `xuid` on join (proves the hook + SteamID); (2) `sm_ban <self> 1 test` from rcon kicks the user; (3) the user reconnects -> **rejected** (banned); (4) `sm_unban <steamid>` -> the user reconnects normally; (5) `bans.json` persists the entry; (6) an expired 1-minute ban lets them back after expiry; (7) `RestartCount=0`, server ticking throughout.

## Risks / decisions

- **`xuid` reliability at ClientConnect (MEDIUM until live).** Steam-auth timing *could* make `xuid` 0/partial pre-auth. The connect log verifies it live; fallback is a post-auth kick (`client_kick` exists). Not blocking the build.
- **Full proof needs a human client.** Same ceiling as SayText2 / damage hooks. In-isolate + install proves everything except "a real reconnect is rejected," which the user tests.
- **Custom reject message deferred** (tier1 `CBufferString::Insert` dlopen risk). Generic engine rejection is sufficient; the reason is still logged server-side.
- **`sm_banip` deferred** — no IP at `ClientConnect`; needs a `client_address` op + engine `addip`/`writeip` enforcement (a separate slice).
- **`ban_check` fails open on panic** — a core bug must never wedge all connections.
