# Slice 6.4 — server-command primitive + `sm_map` (design)

**Goal:** A reusable engine-generic **`Server.command(cmd)`** primitive (run a string at the server
console) + **`Server.isMapValid(map)`**, exercised by **`sm_map <mapname>`** (`ADMFLAG.CHANGEMAP`). One
small op-pair unblocks a whole class of SM admin commands (`sm_map` now; `sm_cvar`/`sm_exec`/`sm_rcon`
later — all "just run a server command"). Continues the base-plugin suite (Slice 6) on the proven 6.1–6.3
spine.

**Status:** design (recommended default, autonomous continuation). Thin vertical slice: one new module
(`@s2script/server`), two ops (`server_command`, `server_map_valid`), one command (`sm_map`),
rcon-verifiable (the map visibly changes).

**Branch:** `slice-6.4-server-command` (off main `7cc8460`).
**Cadence:** subagent-driven, merge-to-main-locally, Docker CS2 live gate.

---

## 1. `@s2script/server` (engine-generic — server control)

Running a console command string / querying map validity are Source 2 facts, not CS2 facts → this is a
core-prelude engine-generic module (`__s2pkg_server`, alongside chat/frame/…), NOT `@s2script/cs2`.

```ts
export declare const Server: {
  /** Run `cmd` at the server console (IVEngineServer2::ServerCommand). Queued; executes next frame. */
  command(cmd: string): void;
  /** Whether `map` is an installed, valid map (IVEngineServer2::IsMapValid). */
  isMapValid(map: string): boolean;
};
```

Backed by two natives over two new ops:
- `__s2_server_command(cmd)` → op `server_command(str)` → `s_pEngine->ServerCommand(str)` (`s_pEngine`
  already held). No-op on null cmd / no op.
- `__s2_server_map_valid(map)` → op `server_map_valid(str) -> i32` → `s_pEngine->IsMapValid(map) != 0`.
  Returns `false` on null / no op / no engine.

## 2. The ops (core + shim; ABI-appended after `client_kick`)

C header (`shim/include/s2script_core.h`, after the `client_kick` typedef/field):
```c
typedef void (*s2_server_command_fn)(const char* cmd);
typedef int  (*s2_server_map_valid_fn)(const char* map);   // 1 = valid, 0 = not
...
    s2_server_command_fn   server_command;
    s2_server_map_valid_fn server_map_valid;
```
Rust mirror (`core/src/v8host.rs` `S2EngineOps`, after `client_kick`, both test literals updated):
```rust
pub type ServerCommandFn  = extern "C" fn(cmd: *const c_char);
pub type ServerMapValidFn = extern "C" fn(map: *const c_char) -> c_int;
...
    pub server_command:   Option<ServerCommandFn>,
    pub server_map_valid: Option<ServerMapValidFn>,
```
Shim (`s2script_mm.cpp`): `s2_server_command` = null/`!s_pEngine` guard → `s_pEngine->ServerCommand(cmd)`;
`s2_server_map_valid` = guards → `return s_pEngine->IsMapValid(map) ? 1 : 0;`. Wire both after
`ops.client_kick`.

The prelude module (mirrors `@s2script/chat`):
```js
var __s2_server = {
  command: function (cmd) { __s2_server_command(String(cmd)); },
  isMapValid: function (map) { return __s2_server_map_valid(String(map)) === 1; },
};
globalThis.__s2pkg_server = { Server: __s2_server };
```

## 3. `sm_map <mapname>` (`ADMFLAG.CHANGEMAP`) — in `@s2script/basecommands`

```ts
Commands.registerAdmin("sm_map", ADMFLAG.CHANGEMAP, (ctx) => {
  const map = ctx.argString.trim().split(/\s+/)[0] || "";
  if (!map) { ctx.reply("Usage: sm_map <mapname>"); return; }
  // Injection guard: we build a "changelevel <map>" string, so the name must be a bare map token.
  if (!/^[A-Za-z0-9_]+$/.test(map)) { ctx.reply("[SM] Invalid map name."); return; }
  if (!Server.isMapValid(map)) { ctx.reply("[SM] '" + map + "' is not a valid map."); return; }
  ctx.reply("[SM] Changing map to " + map + "…");
  Server.command("changelevel " + map);
});
```
`Server` from `@s2script/server`. The `^[A-Za-z0-9_]+$` guard prevents command injection (`a; quit`);
`isMapValid` rejects a well-formed but nonexistent map cleanly (no broken transition). A `changelevel`
triggers a normal map transition → the plugin unloads/reloads at the new map-live (exercises the whole
lifecycle under a real map change).

## 4. Components & boundary

| Concern | Lives in | Engine-generic? |
|---|---|---|
| `server_command` / `server_map_valid` ops + natives; shim `ServerCommand`/`IsMapValid` | core + shim | yes — a command string / map name |
| `@s2script/server` (`Server.command`/`isMapValid`) + `.d.ts` | core prelude + `packages/server` | yes |
| `sm_map` | `@s2script/basecommands` (CS2 plugin, uses `ADMFLAG`) | no |

No CS2 symbol enters `core/src`; both boundary gates stay green.

## 5. Degrade-never-crash

Empty/whitespace map → usage reply. Non-token map name → "Invalid map name" (no injection). Well-formed
but nonexistent map → "not a valid map" (`isMapValid` false), no transition. `server_command`/
`server_map_valid` with no op / null / no engine → no-op / `false`. No path throws into plugin JS.

## 6. Testing & live gate

- **In-isolate (core):** `__s2_server_command` no-ops without the op (no throw); `__s2_server_map_valid`
  returns `false` without the op. (Add to the existing degrade test region.)
- **In-isolate (JS via the prelude / a vm harness):** `Server.command` calls `__s2_server_command` with the
  stringified cmd; `Server.isMapValid` maps `1→true`, `0→false`.
- **CLI/typecheck:** the `@s2script/server` `.d.ts` compiles; `basecommands` passes the 5E.1 typecheck gate.
- **One sniper rebuild** (the two ops + shim).
- **Live gate (de_inferno, rcon):** `sm_map notarealmap` → "not a valid map" (server unchanged);
  `sm_map "a b"` (or `a;quit` form) → "Invalid map name"; `sm_map de_dust2` → the map CHANGES (a follow-up
  `status` shows `de_dust2`, the plugin reloads at map-live, `RestartCount=0`, no crash). Fully verifiable
  with rcon (server console = root; no human client needed).
