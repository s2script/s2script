# Slice 6.13-B — basechat as its own distributable plugin + the `@` chat triggers

**Status:** design (brainstormed + approved 2026-07-05)

**Goal:** Split the SourceMod-parity `basechat` concern out of `examples/basecommands` into its own shippable plugin `@s2script/basechat`, promote the show-activity helper into shared `@s2script/cs2`, and add the SourceMod `@` chat triggers (`@`→say, `@`-in-team→admin-chat, `@@`→psay) over a new engine-generic **raw-chat subscriber** mechanism (`Chat.onMessage`). This continues Slice 6's north star — the SourceMod default-plugin suite, shipped as `@s2script/*`, each an idiomatic separate plugin.

**Non-goals / corrections carried in:** `sm_slap`/`sm_slay` belong in SourceMod's `playercommands` plugin, **not** basecommands and **not** any "basefuncommands" (which is not a real SM plugin) — they stay in `basecommands` for now and move in a later slice. This slice ships exactly one plugin of the suite (`basechat`) plus the mechanism the rest of chat will reuse.

---

## Why one slice

Everything here hangs off two new pieces: (a) a **distribution home** (`plugins/`) so the base suite isn't trapped in `examples/`, and (b) the **raw-chat subscriber** (`Chat.onMessage` + threaded `teamonly`) — the one genuinely new, engine-generic core/shim change (one sniper rebuild). Both `sm_say`'s relocation and all three `@` triggers are then straightforward layering in the new plugin.

---

## Architecture — five units

### Unit 1 — Distribution layout: a top-level `plugins/` directory

The base-plugin suite must be distributable independently of `examples/` (which stays for demos: `demo-plugin`, `entref-*`, `schema-dump`). Create a top-level `plugins/` directory mirroring SourceMod's `plugins/` layout; each subdirectory is one shippable base plugin.

- `git mv examples/basecommands plugins/basecommands` (moves `package.json`, `src/plugin.ts`, and the committed `dist/_s2script_basecommands.s2sp` — the `.s2sp` name derives from the package name `@s2script/basecommands`, so it is unchanged by the move).
- Create `plugins/basechat/` (Unit 4).
- **Typecheck gate:** the only tooling that iterates `examples/` is `scripts/check-plugins-typecheck.sh` (a standalone gate, not wired into CI today). Generalize it to also cover `plugins/`: rename to `scripts/check-plugins-typecheck.sh`, iterate over **both** `examples/*/` and `plugins/*/` (skip dirs without a `package.json`, unchanged per-dir logic calling `typecheckPlugin(d, { packagesDir: 'packages' })`), and update every reference to the old name (grep the repo — CLAUDE.md's gate-suite mention at minimum; there is no CI/Makefile reference).
- **Deploy note (docs/live-gate only):** the live-gate `.s2sp` for basecommands is now built from `plugins/basecommands` (`npx s2script build plugins/basecommands`) and copied from `plugins/basecommands/dist/` into `dist/addons/s2script/plugins/`. `package-addon.sh` and `docker-compose.yml` do **not** reference `examples/` and need no change.

### Unit 2 — Shared show-activity: `Activity.formatSource` in `@s2script/cs2`

SourceMod's `ShowActivity` is a shared native used by more than one plugin (basechat now; kick/slap announcements later), so the helper cannot live inside one plugin. Promote `basecommands`' local `formatActivitySource` into `@s2script/cs2` as `Activity.formatSource`.

- **Runtime (`games/cs2/js/pawn.js`):** define `Activity` inside the IIFE and add `Activity: Activity` to the `__s2pkg_cs2` object literal (currently `{ Pawn, Player, Events, ChatColors }`). It resolves `Admin`/`ADMFLAG` via `__s2require("@s2script/admin")` (game→engine-generic is the allowed dependency direction) and uses `Player.fromSlot`.
- **API surface:** `Activity.formatSource(actorSlot: number, recipientSlot: number): { show: boolean; name: string }` — a **verbatim** port of the current `formatActivitySource` (SM's `FormatActivitySource`):
  - `SHOW_ACTIVITY` flag bits (SM values): `1` kNonAdmins, `2` kNonAdminsNames, `4` kAdmins, `8` kAdminsNames, `16` kRootNames. Default value **`13`** (`1|4|8`), held as a named const in `pawn.js`. (Reading a live `sm_show_activity` cvar is a deferred refinement — keep the hardcoded default so behavior is identical to today.)
  - `actorSlot < 0` ⇒ actor is the server console; generic name defaults to `"Console"`.
  - Generic name (`mode 1`): `"ADMIN"` if the actor has `ADMFLAG.GENERIC`, else `"PLAYER"`, else `"Console"` for the console actor.
  - Decision (unchanged from the current port):
    - Recipient NOT an admin (`!hasFlags(ADMFLAG.GENERIC)`): `show` iff flag `1` or `2`; name = real (`mode 0`) iff flag `2` **or** `recipient == actor`, else generic.
    - Recipient IS an admin: `is_root = recipient hasFlags ADMFLAG.ROOT`; `show` iff flag `4` or `8`, or (`16` and `is_root`); name = real iff flag `8`, or (`16` and `is_root`), or `recipient == actor`, else generic.
- **Types (`packages/cs2/index.d.ts`):** add a hand-written `export declare const Activity: { formatSource(actorSlot: number, recipientSlot: number): { show: boolean; name: string } }` alongside `Player`/`ChatColors`.
- Pure (only admin/name lookups) → unit-testable.

### Unit 3 — The raw-chat subscriber mechanism (engine-generic core + shim; one sniper rebuild)

A plugin subscribes to inbound chat lines and decides whether to suppress the broadcast. This keeps `@`→command routing **in the plugin**, never hardcoded in core.

- **Thread `teamonly` into core.** Widen the FFI + shim boundary:
  - `shim/include/s2script_core.h`: `int s2script_core_dispatch_chat(int slot, const char* text, int teamonly);`
  - `core/src/ffi.rs` `s2script_core_dispatch_chat`: add `teamonly: c_int`, forward to `dispatch_chat(slot, text_str, teamonly != 0)`.
  - `shim/src/s2script_mm.cpp` `Detour_HostSay`: pass the already-received `teamonly` to the call: `s2script_core_dispatch_chat(slot, msg, teamonly ? 1 : 0)`. The suppress path (`if (!suppress) g_origHostSay(...)`) is unchanged.
- **New subscriber mux.** Add `CHAT_MSG_SUBS: RefCell<EventMux<v8::Global<v8::Function>>>` to the `thread_local!` block in `v8host.rs` (mirror `DAMAGE_MUX` — a single un-keyed notify list; use a fixed key such as `""` for `subscribe`/`snapshot`; there is no per-name engine-op to toggle, so ignore the "first subscriber" bool). Teardown: add `CHAT_MSG_SUBS.remove_by_owner(id)` to `unload_plugin` next to the existing `EVENT_MUX`/`DAMAGE_MUX`/`CONFIG_SUBS` teardown (no engine-op on empty).
- **Native + prelude.** Add `__s2_chat_on_message(handler)` (wired via `set_native`, mirroring `__s2_event_subscribe`: extract the handler as `v8::Global`, resolve `owner`/`generation`, `CHAT_MSG_SUBS.subscribe("", owner, gen, handler)`). Extend the `@s2script/chat` prelude object (`v8host.rs` ~575) with `onMessage: function (handler) { __s2_chat_on_message(handler); }`.
- **Dispatch — extend `dispatch_chat(slot, text, teamonly)`** (`v8host.rs` ~1524). Keep the existing `!`/`/` command path exactly: on a matched command, dispatch and `return silent`. **Change:** when no command matched (ordinary chat, `@…`, or an unmatched `!foo`/`/foo`), run the chat subscribers and collapse to a suppress bool — mirror `dispatch_game_event`'s loop:
  1. `let snap = CHAT_MSG_SUBS.snapshot(""); if snap.is_empty() { return false; }`
  2. `HOST.try_borrow_mut()` re-entrancy guard (return `false` if already borrowed).
  3. Per subscriber: `REGISTRY.is_live(owner, gen)` liveness check, per-subscriber `HandleScope`/`ContextScope`, per-handler `TryCatch`; call `handler(slot, text, teamonly)`; interpret the return as a `HookResult` int (`Continue=0`, `Changed=1`, `Handled=2`, `Stop=3`; a non-number / `undefined` ⇒ `Continue`); a thrown handler → `log_warn` and treat as `Continue`.
  4. `suppress = any subscriber returned >= Handled (2)`. `return suppress`.
- **Types (`packages/chat/index.d.ts`):** add `onMessage(handler: (slot: number, text: string, teamonly: boolean) => HookResult | void): void`. Import the `HookResult` type the same way `@s2script/damage`'s `onPre` return is typed (from `@s2script/events`); the runtime `HookResult` is already a prelude global.

### Unit 4 — The `@s2script/basechat` plugin

`plugins/basechat/package.json`: `{ "name": "@s2script/basechat", "version": "0.1.0", "main": "src/plugin.ts", "s2script": { "apiVersion": "1.x" } }`.

`plugins/basechat/src/plugin.ts` — imports `Commands` (`@s2script/commands`), `Chat` (`@s2script/chat`), `Admin`+`ADMFLAG` (`@s2script/admin`), `Player`+`ChatColors`+`Activity` (`@s2script/cs2`), `HookResult` (`@s2script/events`). Three shared helpers + three commands + one chat subscriber.

**Helpers (so a command and its `@` trigger share one code path):**
- `doSay(actorSlot, msg)` — the sender-aware all-chat broadcast, moved verbatim from the old `sm_say`: for each `Player.allConnected()`, `{show, name} = Activity.formatSource(actorSlot, p.slot)`; if `show`, `Chat.toSlot(p.slot, " " + ChatColors.Green + "(ALL) " + name + ": " + ChatColors.White + msg)`.
- `doAdminChat(actorSlot, msg)` — to admins only: for each `Player.allConnected()` whose `Admin.forSlot(p.slot)?.hasFlags(ADMFLAG.CHAT)`, `Chat.toSlot(p.slot, " " + ChatColors.Green + "(ADMINS) " + <actor name> + ": " + ChatColors.White + msg)`; the actor name is the actor's real name (or `"Console"` for `actorSlot < 0`). Console (`actorSlot < 0`) is always allowed to send.
- `doPsay(actorSlot, targetSlot, msg)` — private: `Chat.toSlot(targetSlot, " " + ChatColors.Green + "(private) " + <actor name> + ": " + ChatColors.White + msg)`, and if `actorSlot >= 0 && actorSlot != targetSlot` also echo to the sender `Chat.toSlot(actorSlot, " " + ChatColors.Green + "(private to " + <target name> + ") " + ChatColors.White + msg)`.

**Commands** (all `Commands.registerAdmin(..., ADMFLAG.CHAT, ...)`):
- `sm_say <message>` → guard empty (`ctx.reply("Usage: sm_say <message>")`), else `doSay(ctx.callerSlot, ctx.argString.trim())`.
- `sm_chat <message>` → `doAdminChat(ctx.callerSlot, ctx.argString.trim())` (empty-guarded).
- `sm_psay <target> <message>` → split `ctx.argString` into first token (target) + rest (message); resolve `Player.target(target, ctx.callerSlot)`; require exactly one match (`ctx.reply` on none/ambiguous); else `doPsay(ctx.callerSlot, match.slot, message)`.

**Chat subscriber** — `Chat.onMessage((slot, text, teamonly) => …)`:
- If `text[0] !== '@'` → `return HookResult.Continue` (ordinary chat, shown).
- `const admin = Admin.forSlot(slot); if (!admin || !admin.hasFlags(ADMFLAG.CHAT)) return HookResult.Continue;` — a non-admin's leading `@` is normal chat (SM's conditional consume; the message is NOT suppressed and NOT routed).
- Admin path (all `return HookResult.Handled` to suppress the raw line):
  - `text.startsWith("@@")` → parse `text.slice(2).trim()` as `<target> <message>`; resolve `Player.target(target, slot)`; exactly one → `doPsay(slot, match.slot, message)`, else `Chat.toSlot(slot, <error>)`.
  - else if `teamonly` → `doAdminChat(slot, text.slice(1).trim())`.
  - else → `doSay(slot, text.slice(1).trim())`.

## Data flow

`@msg` typed in all-chat → `Host_Say` detour (`teamonly=false`) → `s2script_core_dispatch_chat(slot, "@msg", 0)` → `dispatch_chat` (not a `!`/`/` command) → `CHAT_MSG_SUBS` loop → basechat's `onMessage` handler → admin check → `doSay(slot, "msg")` → per-recipient `Activity.formatSource` + `Chat.toSlot` → handler returns `Handled` → `dispatch_chat` returns `true` → detour suppresses the raw broadcast. `@msg` in team-chat routes to `doAdminChat`; `@@name msg` routes to `doPsay`.

## Error handling / degrade

- Non-admin `@…` → `Continue` (normal chat), never suppressed, never routed.
- `sm_psay`/`@@` with no/ambiguous target → sender gets an error line; the `@@` line is still consumed (`Handled`).
- A subscriber that throws → `log_warn`, treated as `Continue` (chat shows); a nested re-dispatch is skipped by the `try_borrow_mut` guard (existing pattern).
- `Activity` with a stale/absent slot → `Player.fromSlot`/`Admin.forSlot` already return null-safe; `show=false`/generic name, no crash.
- `teamonly` never crosses as a raw pointer — it is a copied `bool`.

## Testing

- **Unit (node:test):** `Activity.formatSource` across the flag matrix × {actor console/admin/non-admin} × {recipient admin/root/non-admin/self} — the correctness core, no engine needed. (If the pure logic is not directly importable from `pawn.js`, extract the decision into a tiny pure module the runtime and the test both consume, or add a focused fixture test — do not ship it untested.)
- **In-isolate (core):** the natives-degrade asserts for `__s2_chat_on_message` (no engine ops) + a `dispatch_chat` test that a subscriber returning `Handled` yields suppress and `Continue` yields no-suppress (mirror the existing `dispatch_game_event` tests). Keep the existing `dispatch_chat` `!`/`/` tests green.
- **Typecheck gate:** `scripts/check-plugins-typecheck.sh` must pass for `plugins/basecommands` **and** `plugins/basechat` (full strict) — this validates the new `Activity`/`Chat.onMessage` `.d.ts` surface.
- **Live gate (de_dust2, human client — the user):** `sm_say`/`sm_chat`/`sm_psay` via rcon/console; then, as a live client, `@hi` (all-chat) shows `(ALL) <name>: hi` to everyone, `@hi` in team-chat shows `(ADMINS) <name>: hi` to admins only, `@@<name> hi` privately messages the target; a non-admin's `@hi` shows as ordinary chat; `RestartCount=0`, server ticking.
