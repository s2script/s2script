# Slice 6.5 — admin-utility commands (`sm_who` / `sm_rcon` / `sm_exec`) design

**Goal:** Three canonical SourceMod admin commands, all built on already-proven primitives (`Player.allConnected`, `Admin.forSlot`, `Server.command`) — **no new engine op, no core/shim change, no sniper build.** Rounds out the base admin toolkit cheaply while the RE-heavy capabilities (cvar get/set, slay, damage pre-hooks) stay queued.

**Status:** design (recommended default, autonomous continuation). JS-only, rcon/bot-verifiable.

**Branch:** `slice-6.5-admin-utils` (off main `458a141`).
**Cadence:** direct implementation (a single-file plugin change) + a review pass + Docker live gate + merge-to-main-locally. (No SDD subagent fan-out — the change is one small file with no framework surface.)

---

## 1. `sm_who` (`ADMFLAG.GENERIC`)

Lists every connected player (SM's `sm_who`): for each `Player.allConnected()` — `#userid name slot=N steamid admin=<yes(flags=0xNN)/no>`. Admin status via `Admin.forSlot(slot)` (`AdminInfo | null`; `null` → not an admin — bots read steamid `"0"` → never admin). `GENERIC` is the SM flag for `sm_who`. Reply is multi-line (one `ctx.reply` per player + a header), routed to the caller (console for rcon).

## 2. `sm_rcon <command>` (`ADMFLAG.RCON`)

A deliberate full server-command passthrough (SM's `sm_rcon`): `Server.command(ctx.argString.trim())`. **No sanitize** — `ADMFLAG.RCON` is the highest-trust flag and the command IS "run arbitrary server commands." (The Source console's own `;`-split applies, identical to SM — an admin with RCON can already run anything.) Empty → usage. Reply "[SM] Command sent."

## 3. `sm_exec <cfgfile>` (`ADMFLAG.CONFIG`)

Executes a server config (SM's `sm_exec`): `Server.command("exec " + file)`. Because we concatenate into a command string, the filename is sanitized: `^[A-Za-z0-9_./-]+$` (allows `foo.cfg`, `sub/foo.cfg`, `-`/`_`; rejects `;`, whitespace, other shell/console chars) AND reject `..` (no path traversal). Empty / invalid → usage / "Invalid config name." Reply "[SM] Executing <file>."

## 4. Boundary & degrade

All three live in `@s2script/basecommands` (CS2 plugin) using `@s2script/commands` (`registerAdmin` + `ctx`), `@s2script/admin` (`ADMFLAG` + `Admin.forSlot`), `@s2script/cs2` (`Player.allConnected`), `@s2script/server` (`Server.command`). No core/shim/`packages/*` change. Degrade: empty/invalid args → a usage/"invalid" reply; `Server.command` is already null-safe; `Admin.forSlot` returns null for non-admins/bots. Nothing throws.

## 5. Testing & live gate

- **Typecheck (5E.1 gate):** `basecommands` compiles (`scripts/check-examples-typecheck.sh`).
- **Build the `.s2sp`; NO sniper** (JS-only). Deploy = drop the `.s2sp` + restart.
- **Live gate (de_dust2/de_inferno, rcon, `bot_quota 2`):**
  - `sm_who` → lists the 2 bots (`#uid name slot=N steamid=0 admin=no`).
  - `sm_rcon bot_add` (or `sm_rcon say hi`) → the effect happens (a 3rd bot / a chat line); reply "Command sent." Avoid `;` in the test string.
  - `sm_exec notacfg;quit`-style is NOT tested (console `;`-split); `sm_exec bad name` → "Invalid config name"; `sm_exec gamemode_competitive` (a real cfg) → "Executing …" + no crash. (An exec of a nonexistent cfg is a harmless engine no-op.)
  - `RestartCount=0`, no crash.

## 6. Out of scope (do not build ahead)

`sm_cvar` get/set (needs a cvar-value read primitive — a `ConVarData` type-union read at gamedata offsets, a moderate RE slice); `sm_slay`/`sm_burn`/`sm_freeze` (need a bot-safe player-effect/kill — engine `ClientCommand` on a fake client risks the `ClientPrintf` null-netchannel crash, so a game-function call is required); non-event pre-hooks (`OnTakeDamage`); `sm_reloadadmins` command; `sm_kickall`. Each is a separate, RE-bearing slice. Note as TODOs and stop.
