# Slice 4.5 — Inter-Plugin Typed Interfaces ("two plugins talk")

**Status:** design approved, ready for writing-plans.
**Branch:** `slice-4.5-inter-plugin-interfaces` (off `main`, which has Slices 0–4 merged).
**Prior art in-repo:** `docs/superpowers/specs/2026-06-30-slice-4-plugin-lifecycle-design.md`.

---

## 1. Goal — the closing thread

One producer plugin **publishes** a typed, versioned interface; a second consumer plugin
**imports** it, **calls a method** on it, and **receives an event** it forwards — and the whole thing
tears down cleanly in reverse-dependency order. Proven end-to-end on the live Docker CS2 server with
two real `.s2sp`s, exactly as CLAUDE.md's "Cross-plugin comms are typed, versioned interfaces"
guardrail describes: *methods = natives, events = forwards, one object, semver-governed; hard deps
return a proxy that throws on producer-unload, optional deps return `Interface | null`; all imports
ledgered; unload resolves in reverse-dependency order.*

This is the §2.9 slice. It builds directly on Slice 4's machinery and adds no new engine touchpoints
beyond what the contract needs.

## 2. What we build on (Slice 4, all merged)

- **`PLUGINS`** — per-plugin `Global<Context>` + captured `module.exports`, keyed by plugin id.
- **`REGISTRY` / `PluginLedger`** (`core/src/plugin.rs`) — records `Hook`/`Timer`/`Job` resources;
  `teardown_order()` returns them reversed. Generation counter; `is_live(id, gen)` async-liveness.
- **`__s2require`** (`s2require` native) — today maps `@s2script/std`→`__s2pkg_std`,
  `@s2script/cs2`→`__s2pkg_cs2` on the calling context's global; any other specifier → JS `null`.
- **`load_plugin_js` / `unload_plugin`** — CJS wrapper eval, `onLoad`/`onUnload` in the plugin's own
  context, reverse-ledger teardown, per-plugin context dispose.
- **`current_plugin(scope)`** — reads the calling context's `PluginId` slot (correct across the
  microtask checkpoint).
- **`loader::Manifest`** — parses `{id, version, apiVersion}` today (the `apiVersion` same-major check
  landed in Slice 4 cleanup); the CLI already writes `pluginDependencies` + `publishes` into
  `manifest.json` but the loader ignores them.
- **CLI `buildPlugin`** — esbuild bundle (`--format=cjs`, `external: ["@s2script/std","@s2script/cs2"]`),
  derives `manifest.json`, zips the `.s2sp`.

## 3. Decisions locked during brainstorming

1. **Scope = methods + one event forward.** Prove *both* directions of the contract (consumer→producer
   call and producer→consumer forward), because both exercise the ledger / reverse-dependency teardown.
2. **Marshalling = structured-copy.** Every value crossing the boundary (method args, return, event
   payload) is serialized and rebuilt in the target context — no shared object identity ever crosses.
   This enforces "never expose a raw cross-plugin reference across time" *structurally*, not by
   convention. Faithful to SourceMod's precedent (cells by value, strings/arrays copied at the native
   boundary; complex objects only ever crossed as core-mediated Handles — never raw live objects).
   Mechanism for this slice: a **JSON string carrier** — stringify in the source context → a neutral
   Rust `String` → parse in the target context (the spec's stated acceptable fallback; `ValueSerializer`
   is a later richer-types upgrade). Values JSON cannot represent (cyclic reference, BigInt) fail the
   copy → a named throw (`InterfaceValueNotSerializable`); functions/`undefined` serialize to `null`
   (lossy but still identity-safe — no live reference crosses). Callbacks are passed via events, never
   as args, so the null-coercion is not a limitation for the contract.
3. **EntityRef on the wire = excluded from 4.5.** The handle/EntityRef wrapper system is Slice 5; when
   it lands, entity refs on the wire marshal as that same `EntityRef`/`T | null` type. No throwaway
   stub now. The 4.5 wire carries plain serializable data only.

## 4. Architecture — Approach A: core-mediated registry + name-resolved ES-Proxy handles

Core owns a single **`InterfaceRegistry`** (new module `core/src/interfaces.rs`, engine-generic, no CS2
identifiers, pure logic + unit-tested). A published interface is an entry keyed by name:

```
InterfaceEntry {
    version: String,            // concrete version the producer published, e.g. "1.0.0"
    producer_id: String,        // owning plugin id
    producer_gen: u64,          // owning plugin generation (async-liveness tag)
    method_names: Vec<String>,  // enumerated function props of the published impl
    // method Global<Function>s + subscriber Global<Function>s live in v8host, keyed by name
}
```

The consumer's handle is an **ES `Proxy`** built in JS by the `@s2script/std` prelude
(`makeIfaceProxy(name)`), *not* an enumerated object. Every property access yields a native-backed
caller, so a **hard** dependency can be `require`d *before* the producer has published — the call
throws until the producer is present. This removes any dependence on load order (topo *load* order
stays deferred) and makes producer hot-reload auto-recover: the proxy re-resolves the registry by
name on every call.

## 5. API surface (all engine-generic → `@s2script/std`)

**Producer:**
```js
const greeter = publishInterface("@demo/greeter", "1.0.0", {
  greet(slot) { return `hello, player ${slot}`; },   // method = native
});
// ... later, on a tick / concommand / event:
greeter.emit("greeted", slot);                         // event = forward
```
`publishInterface(name, version, impl)` returns an **emit-handle** `{ emit(event, payload) }`. The impl
object's own **function** properties are the method set (enumerated at publish).

**Consumer:**
```js
const g = require("@demo/greeter");        // hard dep → always a Proxy
console.log(g.greet(42));                   // cross-context call, structured-copied
g.on("greeted", (slot) => { /* ... */ });   // subscribe to a forwarded event
g.off("greeted", handler);                  // (optional) unsubscribe
```
For an **optional** dep, `require` returns the Proxy if the producer is currently published (and
version-compatible), else `null` — the consumer null-checks.

`require` is unchanged for `@s2script/std` / `@s2script/cs2`. The CJS `require` handed to plugins
becomes a prelude `__s2_require(name)` that dispatches: builtin package → existing native; any other
name → `resolveInterface(name)` (Proxy for hard, Proxy-or-`null` for optional, `null` for undeclared).

## 6. Core components

- **`core/src/interfaces.rs` (new):** `InterfaceRegistry` + pure logic — `publish`, `lookup`, `remove`,
  subscriber add/remove, **semver satisfaction** (minimal caret/major, same style as the Slice-4
  `apiVersion` check), dep-kind categorization, and the **reverse-dependency ordering** (Kahn over the
  consumer→producer edges). No V8, no CS2 ids. Unit-tested in an inline `#[cfg(test)] mod`.
- **`core/src/v8host.rs` (modify):** seven new `__s2_*` natives — `iface_publish`, `iface_emit`,
  `iface_call`, `iface_on`, `iface_off`, `iface_dep_kind`, `iface_is_published` — each
  `catch_unwind`-wrapped and degrade-never-crash. The `Global<Function>` stores (method + subscriber
  handles) live here, keyed by interface name. Prelude additions: `publishInterface`,
  `makeIfaceProxy`, `resolveInterface`, `__s2_require` dispatch. `set_plugin_imports(id, imports)` so
  `iface_dep_kind` can categorise `require` for the current plugin. Teardown hooks in `unload_plugin`.
- **`core/src/plugin.rs` (modify):** new `Resource` variants `Interface(String)` (producer-owned),
  `EventSub(u64)` (consumer-owned), `Import(String)` (consumer→producer edge, for ordering). New
  `record_interface` / `record_event_sub` / `record_import`. All flow through `teardown_order()`.
- **`core/src/loader.rs` (modify):** `Manifest` gains `#[serde(default)] pluginDependencies` +
  `optionalPluginDependencies` (both `HashMap<String,String>`); the loader hands each plugin's
  `{name → (range, kind)}` import map to core via `v8host::set_plugin_imports(id, imports)` **before**
  `load_plugin_js` runs the module eval (so `require` can categorise during eval), cleared on unload.
- **`packages/cli/src/build.ts` (modify):** emit `optionalPluginDependencies` into `manifest.json`;
  mark **every** inter-plugin dep name (all keys of both dep maps) as an esbuild `external`, in
  addition to the two builtin packages — otherwise esbuild tries to bundle `@demo/greeter` and fails.
- **`packages/std/index.d.ts` (modify):** add `publishInterface` + the `require`/proxy types.

No new C-ABI is required: import metadata flows through the loader, which is already in-core.

## 7. Data flow

- **publish** → `iface_publish` records the registry entry keyed by name, reflects the impl's function
  props into method `Global`s, ledgers `Interface(name)` on the producer (`current_plugin`).
- **resolve / require** → prelude `resolveInterface(name)` calls `iface_dep_kind(name)` (reads the
  current plugin's import map). `none` → `null`; `optional` → `iface_is_published(name)` ? Proxy :
  `null`; `hard` → always the Proxy. Building the Proxy also ledgers `Import(name)` on the consumer.
- **call** `g.greet(x)` → the Proxy's `get` trap returns `(...args) => __s2_iface_call(name,"greet",args)`.
  `iface_call` **re-resolves** the entry by name, checks the version range, structured-copies args
  consumer→producer, enters the producer context, calls the method `Global`, structured-copies the
  return back. Absent/stale/mismatch → throw (see §8).
- **emit** `greeter.emit("greeted", p)` → `iface_emit` dispatches to every **live** subscriber
  (`is_live(consumer_id, gen)`), entering each consumer context with a structured-copied payload.
- **on** `g.on("greeted", fn)` → `iface_on` registers the consumer's handler `Global` in the
  producer's subscriber list, ledgers `EventSub(id)` on the consumer tagged `(id, gen)`.

## 8. Error handling & invalidation (degrade-never-crash)

Every native is `catch_unwind`-wrapped. Failures throw a **named** JS error in the consumer's context;
uncaught, it bubbles to the enclosing dispatch `TryCatch` (the `OnGameFrame` handler, `onLoad`, …) →
logged WARN, that plugin degrades, the framework runs on.

- **Hard proxy, producer absent/stale** (never loaded, unloaded, or reloaded-away) →
  `InterfaceUnavailable("@demo/greeter")`.
- **Optional dep** → `require` returns `null` when the producer isn't currently published+compatible;
  a held optional proxy behaves like a hard one afterward (the documented pattern is re-`require` +
  null-check).
- **Semver mismatch** — the producer's published version must satisfy the consumer's declared range →
  else hard call throws `InterfaceVersionMismatch`; optional → treated as absent (`null`).
- **Non-serializable arg/return/payload** — a value JSON cannot represent (cyclic reference, BigInt)
  fails the copy → `InterfaceValueNotSerializable`. (Functions/`undefined` coerce to `null` per JSON
  semantics — lossy but identity-safe; callbacks travel via events, not args.)

## 9. Teardown & reverse-dependency ordering

New ledger resources auto-recorded, torn down in reverse acquisition order:

- **Producer unload** → each `Interface(name)` → remove the registry entry, drop the method `Global`s,
  drop the subscriber list. Consumers holding proxies keep running; their next call re-resolves →
  absent → throws (this *is* "hard dep → proxy throws on producer-unload").
- **Consumer unload** → each `EventSub(id)` → remove it from the producer's subscriber list + drop the
  handler `Global`; each `Import(name)` → drop the edge. The producer is unaffected.
- **Async-liveness interaction** — `EventSub` handlers carry the `(id, gen)` tag (same mechanism as
  async resolvers), so an `emit` after a consumer unloads/reloads reaches nobody; a lingering entry is
  dropped by the liveness check.
- **Reverse-dependency ordering** — for an unload-*all* / shutdown cascade, importers (consumers)
  unload **before** producers, so a consumer's `onUnload` can still call producer methods. A minimal
  edge-based ordering pass (unload a plugin only once all its importers are gone). The 2-plugin slice
  proves it; the general N-plugin case is the identical algorithm. (Full topo *load* order stays
  deferred — hard proxies tolerating an absent producer is what removes the need for it.)

## 10. Front-loaded spike (mirrors Slice 4)

Validate the two real unknowns before the build, in a throwaway spike, findings written to a dated
`docs/superpowers/specs/…-slice-4.5-spike-findings.md`:

1. **V8 `ValueSerializer`/`ValueDeserializer` round-trip across two contexts in one isolate** via
   rusty_v8 — confirm a value serialized in context A deserializes into context B with no shared
   identity, and that a function/live-object fails the serialize (the enforcement we rely on).
2. **A prelude-built ES `Proxy` whose traps call natives** that invoke a producer's `Global<Function>`
   — confirm the `get`-trap → native-call → cross-context `Function::call` path works and returns.
   (Cross-context *calls* are already proven by Slice 4's onLoad/onUnload, so this is lower-risk; the
   Proxy trap wiring is the new part.)

If either proves unworkable, revise before the load-bearing tasks.

## 11. Testing & acceptance

**Cargo-unit-testable (the bulk):**
- `plugin.rs` — new `Resource` variants tear down in reverse order; `record_*`.
- `interfaces.rs` — publish/lookup/remove, subscriber add/remove, semver satisfaction, dep-kind
  categorization, reverse-dependency ordering. Pure, exhaustive.
- **In-isolate `v8host` tests** (like `frame_tests`): two contexts — producer publishes, consumer
  resolves + `greet()` structured-copy round-trips a value; producer `emit` → consumer handler fires;
  producer unload → consumer call throws `InterfaceUnavailable`; consumer unload → subscriber removed
  (later `emit` reaches nobody); reverse-dep order → consumer `onUnload` calls the producer, unload-all,
  assert it still worked; non-serializable arg → `InterfaceValueNotSerializable`.
- `loader.rs` — `Manifest` parses both dep maps.
- **CLI** — a two-plugin fixture asserts `manifest.json` carries both dep maps AND the inter-plugin
  name is marked external (absent from the bundled `plugin.js`).

**Live-only (Docker CS2 gate) — the acceptance thread:**
1. Build + drop `examples/greeter-plugin` (producer) and `examples/greeter-consumer` (consumer).
2. Load both → consumer logs a greeting from `g.greet(...)` **and** the forwarded `greeted` event.
3. Unload the producer → the consumer's next call throws → caught → WARN, consumer degrades, server
   keeps ticking (no crash).
4. Reload the producer → the consumer recovers automatically (proxy re-resolves).
5. Unload the consumer → clean teardown, no leak, producer keeps running.

**Acceptance criteria:**
1. `cargo test -p s2script-core` green (existing 51 + the new tests); both boundary gates green; CLI
   `npm test` green; sniper build OK.
2. `s2script build` produces loadable producer + consumer `.s2sp`s with correct manifests + externals.
3. The five live-gate steps pass on the Docker CS2 server.
4. README documents the runbook + acceptance; CLAUDE.md "Current state" updated (Slice 4.5 done,
   focus → Slice 5).

## 13. Scope & deferrals

**Scope:** one coherent vertical slice — no decomposition needed.

**Reaffirmed deferrals (do NOT build ahead):** the `tsc` typecheck gate; the handle/`EntityRef`
wrapper system **and interface `.d.ts` codegen** (Slice 5); the engine-generic `@s2script/std` breadth

`publishes` in the manifest is parsed-but-optional at runtime for this slice (the `publishInterface`
call is authoritative); manifest-vs-runtime publish cross-validation is a future guardrail, noted as a
TODO, not built.

## 14. Global constraints (bind every task)

- **Core stays engine-generic.** No CS2 identifiers, no `include_str!`/`include_bytes!`, no `games/`
  references in `core/src`. `interfaces.rs` and every new native are engine-generic; both CI boundary
  gates (`check-core-boundary.sh`, `test-boundary-nameleak.sh`) must stay green. The inter-plugin API
  lives in `@s2script/std` (generic), never in `@s2script/cs2`.
- **Degrade-never-crash.** Every native `catch_unwind`-wrapped; every failure is a named WARN /
  named JS throw, never a global crash. No panic crosses the FFI boundary.
- **The ledger is the teardown authority.** Interfaces, event subscriptions, and import edges are
  auto-ledgered; teardown walks the ledger and does not depend on plugin cleanup code running.
- **Structured-copy at the boundary.** No V8 value crosses a plugin boundary by shared identity; args,
  returns, and payloads are serialized copies. Non-serializable → named throw.
- **Naming convention (locked Slice 4):** PascalCase events + types, camelCase functions + properties
  (`publishInterface`, `emit`, `greet`, `on`/`off`).
- **cdylib test constraint:** unit tests are inline `#[cfg(test)] mod` in the source file (no
  `core/tests/` — can't link).
- **Contracts are versioned.** The producer publishes a concrete version; the consumer declares a
  range; incompatibility fails fast (throw / null), never silent drift.
