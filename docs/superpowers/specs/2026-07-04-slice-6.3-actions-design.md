# Slice 6.3 — target resolution + sm_kick + sm_slap (design)

**Goal:** The first real player *actions* + SourceMod target-string resolution. A target helper turns
`#userid`/name/`@all`/`@me` into `Player[]`; `sm_kick` disconnects them; `sm_slap` damages + knocks them
back. Completes the base-command trio's action side (6.1 spine → 6.2 admin → **6.3 actions**).

**Status:** design approved. Core target set (`#userid`, exact/partial name, `@all`, `@me`); `sm_slap` =
always-damage + best-effort velocity knockback. One slice, fully bot/rcon-verifiable.

**Branch base:** `main` (… + 6.2 merged).
**Cadence:** subagent-driven, merge-to-main-locally, Docker CS2 live gate.

---

## 1. Target resolution (`Player.target`, CS2 layer)

```ts
Player.target(pattern: string, callerSlot: number): Player[];
```

Resolves a SourceMod target string to the matching connected players (empty array = no match). It needs
the CS2 `Player` model (userid, name), so it lives on `Player` (games/cs2/js/pawn.js + packages/cs2), not
core. Supported selectors (the core set):

- **`#<userid>`** → `[Player.fromUserId(userid)]` (empty if none).
- **`@all`** → `Player.allConnected()`.
- **`@me`** → `[Player.fromSlot(callerSlot)]` (empty if `callerSlot < 0` — the server console has no "me").
- **name** (anything else) → connected players whose `name` contains the pattern (case-insensitive). An
  exact (case-insensitive) name match returns just that one; otherwise all partial matches (the command
  decides single-vs-multi — `sm_kick` on multiple partials replies "multiple matches", `sm_slap` acts on
  all). Team/`@bots`/`@alive` filters are a deferred follow-up.

## 2. `sm_kick <target> [reason]` (`ADMFLAG.KICK`)

Resolve the target → `player.kick(reason)` each. **The kick action:** a new engine-generic
`client_kick(slot, reason)` op (ABI-appended after `client_steamid`) → the shim calls
`IVEngineServer2::KickClient(CPlayerSlot(slot), reason, NETWORK_DISCONNECT_KICKED /*=39*/)` (`s_pEngine`
already held). CS2 `player.kick(reason = "Kicked by admin")` wraps the `__s2_client_kick` native. Reply
the count kicked; empty target → "no matching players".

## 3. `sm_slap <target> [damage=0]` (`ADMFLAG.SLAY`)

Resolve the target → for each player's pawn:
- **Damage (reliable):** `pawn.health = Math.max(1, pawn.health - damage)` via the generated `health`
  setter (which `writeInt32` + `notifyStateChanged`s). A direct health write — proper damage-with-events
  (`OnTakeDamage`) is the deferred non-event-hooks work; slap keeps the player alive (clamp ≥ 1; `sm_slay`
  is a later lethal command).
- **Knockback (best-effort):** `pawn.setVelocity(x, y, z)` — read the current `m_vecAbsVelocity` (3 floats,
  the same direct field the 5C.3 `absVelocity` read uses), add a slap impulse (a horizontal nudge +
  ~300 up, SM-style), and write it back via 3 `writeFloat32` at the field offset (+`offset+4`/`+8`) +
  `notifyStateChanged` on the field. **A new CS2 `pawn.setVelocity(x,y,z)` helper** (JS, no new native —
  reuses `writeFloat32`). If a direct velocity write doesn't "stick" live (physics resets it next tick),
  the damage still applies and knockback becomes a follow-up (may need an engine impulse call); the slice
  does not depend on knockback landing.

## 4. Components & boundary

| Concern | Lives in | Engine-generic? |
|---|---|---|
| `client_kick` op (ABI-append) + `__s2_client_kick` native; shim `KickClient` | core (`v8host.rs`) + shim + gamedata | yes — `(slot, string)` |
| `Player.target` (selectors), `Player.kick`, `pawn.setVelocity` | games/cs2/js/pawn.js + packages/cs2 | no — CS2 player/pawn |
| `sm_kick`/`sm_slap` | `@s2script/basecommands` (CS2 plugin) | no |

No CS2 symbol in `core/src` (the kick op is `(slot, reason)`); both boundary gates stay green.

## 5. Degrade-never-crash

Empty/invalid target → empty `Player[]` → the command replies "no matching players" (no action). A stale
`Player`/`Pawn` (serial-gated) → its accessors read null / writes no-op → skipped (never a crash).
`client_kick` with no op / a bot / an invalid slot → no-op. `pawn.setVelocity` on a null/stale pawn →
no-op. `#userid` with no match → empty. No path throws into plugin JS.

## 6. Testing & live gate

- **In-isolate (vm, CS2 JS):** `Player.target` selectors (`#userid`→fromUserId; `@all`→allConnected;
  `@me`→caller / empty for console; exact name → one; partial → all; no-match → empty) with a stubbed
  Player set; `Player.kick` calls `__s2_client_kick(slot, reason)`; `pawn.setVelocity` writes 3 floats +
  notifyStateChanged (via a stubbed EntityRef). (Mirror the 5C.2 player-model vm harness.)
- **In-isolate (core):** the `__s2_client_kick` native degrades to no-op without the op.
- **One sniper rebuild** (the `client_kick` op + shim).
- **Live gate (de_inferno, rcon, `bot_quota 2`):** `sm_kick @all` → both bots disconnect
  (`allConnected` → 0; the bots gone from the log). `sm_kick #<userid>` → that one bot. `sm_slap <name> 10`
  → read the bot's `pawn.health` before/after (dropped by 10) + `pawn.absVelocity` changed (best-effort;
  log whether it stuck). `sm_slap @all` → all bots damaged. A non-existent target → "no matching players".
  `RestartCount=0`, no crash. Fully verifiable with bots (no human client needed).
