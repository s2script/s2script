# Slice 6.14: `playercommands` (sm_slap + sm_slay + sm_rename) — Implementation Plan

> **For agentic workers:** mechanical tasks (1, 4) via superpowers:subagent-driven-development; the RE spike + core/shim primitives (2, 3) are engine bring-up — driven directly with a tight sig/build/observe loop (like Slice 6.6). Steps use `- [ ]`.

**Goal:** Ship `@s2script/playercommands` = `sm_slap` (moved from basecommands) + `sm_slay` (CommitSuicide by signature) + `sm_rename` (a new entity string-write primitive + a netvar write).

**Architecture:** One trivial move, one RE spike, one new primitive; a single sniper rebuild carries both new ops (`pawn_commit_suicide`, `ent_write_string`) + the CommitSuicide signature. Core stays engine-generic. Spec: `docs/superpowers/specs/2026-07-05-slice-6.14-playercommands-design.md`.

## Global Constraints

- **Core engine-generic:** `check-core-boundary.sh` + `test-boundary-nameleak.sh` green — no CS2 names in `core/src`. The CommitSuicide **signature** lives in `gamedata/core.gamedata.jsonc`; `m_iszPlayerName`'s offset is **live-resolved** via `__s2_schema_offset`, never baked; both new ops take slot/entity-ref + generic args.
- **RE doctrine:** the CommitSuicide address is **self-resolved on our pinned `libserver.so`** via a `resolve:"direct"` prologue signature validated UNIQUE at the 6.9 gate — NOT a borrowed vtable index (that is the exact `slice-6.8-slay`/`a085d5a` failure). Other frameworks' gamedata = hints only.
- **Plugins are pure ESM.** `ADMFLAG.SLAY` gates all three commands. New ABI ops are APPENDED after the last existing op (C header + Rust mirror + both test literals, ABI order preserved).
- **Degrade-never-crash:** stale serial / unresolved sig / null pawn → op no-ops; `sm_slay` replies "unavailable" rather than silently doing nothing.
- **Git:** branch `slice-6.14-playercommands`. `git commit -F -` + heredoc (NEVER `-m`, NEVER backticks). Body ends with `Claude-Session: https://claude.ai/code/session_01G8RQTQTdmeczdE6oihp8Pn`.
