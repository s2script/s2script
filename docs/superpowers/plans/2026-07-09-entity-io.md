# Entity I/O — Implementation Plan (fire inputs + hook outputs)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (this project executes via the `Workflow` tool). Steps use `- [ ]` checkboxes.

**Goal:** `EntityRef.acceptInput(input, value?, activator?, caller?, delay?)` (fire inputs via `AddEntityIOEvent`) + `Entity.onOutput(classname, output, handler)` (hook outputs via a `FireOutputInternal` detour; handler may return `HookResult` to suppress).

**Architecture:** One new `S2EngineOps` op (`entity_fire_input`, core→shim) + one core FFI export (`s2script_core_dispatch_output`, shim-detour→core) + core natives for the output-mux subscribe/unsubscribe. The shim fires inputs via the validated `AddEntityIOEvent` (variant_t from the SDK) and detours `FireOutputInternal` (the `shim/src/detour.cpp` engine from the 6.6 damage hook), dispatching **synchronously** into core (the damage/event pre-hook pattern) so a handler can block. All engine-generic.

**Tech Stack:** Rust (core, rusty_v8), C++ (shim, hl2sdk), JavaScript (prelude), TypeScript (`.d.ts`).

## Global Constraints

- **Boundary (both gates green):** everything is Source 2 (`CEntityInstance`/`CEntityIOOutput`/`variant_t`) — `EntityRef.acceptInput` + `Entity.onOutput` are engine-generic `@s2script/entity`; the op + FFI export take `(idx, serial, string, …)` — NO CS2 schema names in core. The sigs + the `m_pDesc`/`m_pName` offsets are gamedata/SDK. Run `bash scripts/check-core-boundary.sh` + `bash scripts/test-boundary-nameleak.sh`.
- **ABI discipline:** `entity_fire_input` APPENDED after the current last op — never reordered — identical across the C header, Rust mirror, BOTH in-isolate test op-structs, and the shim `ops.` assignment. The `s2script_core_dispatch_output` FFI export mirrors the existing `s2script_core_dispatch_game_event` export shape.
- **No raw pointer to JS:** activator/caller cross the detour→core boundary ONLY as packed `CEntityHandle` ints (shim `GetRefEHandle().ToInt()`), decoded + `entity_resolve_ptr`-validated core-side before `build_entity_ref`. The `CEntityIOOutput`/`CVariant`/entity pointers stay shim-side; the output value crosses as a string.
- **Degrade-never-crash:** every native/export `catch_unwind`-wrapped; `entity_fire_input` returns false on unresolved op/stale/absent entity system; the detour is `s_p*`-null-guarded and falls through to the original on any failure; the synchronous output dispatch is `try_borrow_mut`-guarded (a nested output-fire from a handler skips gracefully — the documented `Events.fire` limitation).
- **RE doctrine:** `AddEntityIOEvent` + `FireOutputInternal` sigs are borrowed HINTS, re-scanned UNIQUE in our binary (already validated 2026-07-09 @0x2128170 / @0x2132a60). `variant_t` is the SDK type (`datamap.h`), not hand-RE'd.
- **Tests serial:** `.cargo/config.toml` `RUST_TEST_THREADS=1`; cargo from `core/`.
- **Commits:** end with `Claude-Session: https://claude.ai/code/session_01G8RQTQTdmeczdE6oihp8Pn`; NEVER backticks in `git commit -m` — use `-F -` heredoc. Commit LOCALLY, do NOT push.
- **Confirmed ABIs (CSSharp C++):**
  - `AddEntityIOEvent(CEntitySystem* sys, CEntityInstance* target, const char* input, CEntityInstance* act, CEntityInstance* caller, variant_t* value, float delay, int outputID, void*=null, void*=null)`. `sys` = the `CGameEntitySystem*` the shim already resolves (GameResourceService + 0x50). Extract: output name `this->m_pDesc->m_pName`; source class `caller->GetClassname()`; activator/caller handles; value→string (read `CVariant::m_type` + the typed field, `natives_cvariant.cpp` pattern).

```cpp
using AddEntityIOEventFn = void (*)(void* sys, CEntityInstance* target, const char* input, CEntityInstance* act, CEntityInstance* caller, variant_t* value, float delay, int outputID, void*, void*);
static AddEntityIOEventFn s_pAddEntityIOEvent = nullptr;

static int Shim_EntityFireInput(int idx, int serial, const char* input, const char* value,
                                int actIdx, int actSerial, int callerIdx, int callerSerial, float delay) {
    if (!s_pAddEntityIOEvent || !input) return 0;
    CEntityInstance* target = ResolveEntityBySerial(idx, serial);
    if (!target) return 0;
    void* sys = GetGameEntitySystem();          // the already-resolved CGameEntitySystem*
    if (!sys) return 0;
    CEntityInstance* act    = (actIdx    >= 0) ? ResolveEntityBySerial(actIdx, actSerial) : nullptr;
    CEntityInstance* caller = (callerIdx >= 0) ? ResolveEntityBySerial(callerIdx, callerSerial) : nullptr;
    variant_t v(value ? value : "");            // SDK variant_t(const char*)
    s_pAddEntityIOEvent(sys, target, input, act, caller, &v, delay, 0, nullptr, nullptr);
    return 1;
}
```

```cpp
static FireOutputInternalFn s_pFireOutputInternal = nullptr, s_origFireOutputInternal = nullptr;
static void Hook_FireOutputInternal(CEntityIOOutput* pThis, CEntityInstance* act, CEntityInstance* caller,
                                    const CVariant* value, float delay, void* u1, char* u2) {
    int result = 0; // Continue
    if (pThis && pThis->m_pDesc && pThis->m_pDesc->m_pName) {
        const char* outputName = pThis->m_pDesc->m_pName;
        const char* cls = caller ? GetEntClassname(caller) : "";
        int actH    = act    ? act->GetRefEHandle().ToInt()    : -1;
        int callerH = caller ? caller->GetRefEHandle().ToInt() : -1;
        char valbuf[256]; CVariantToString(value, valbuf, sizeof valbuf);   // "" if none
        result = s2script_core_dispatch_output(cls, outputName, actH, callerH, valbuf, delay);
    }
    if (result >= 2 /*Handled|Stop*/) return;   // suppress: skip the original (do NOT call it)
    s_origFireOutputInternal(pThis, act, caller, value, delay, u1, u2);
}
```

```ts
  e.spawn();
  const before = e.isValid();
  e.acceptInput("Kill");
  ctx.reply("[entityio] Kill fired; valid before=" + before + " (gone next tick)");
```
