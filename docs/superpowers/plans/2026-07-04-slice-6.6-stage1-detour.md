# Slice 6.6 Stage 1 — MinHook detour + read-only damage log Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Prove — LIVE — that we can detour `CBaseEntity::DispatchTraceAttack` and read the damage on a real bullet hit. The riskiest thread first; the `damage_mux` / JS API / block+modify (Stages 2–4) come only after this fires live.

**Architecture:** Vendor MinHook (self-contained x64 detour lib) into the shim; add a `DispatchTraceAttack` byte-signature to the gamedata `.signatures` section; resolve it with the existing 5D.2 `FindPattern`/`LoadSignatures`; `MH_CreateHook` it; a read-only detour handler logs the victim ptr + candidate damage reads and always calls the original via the trampoline.

**Tech Stack:** C++ (Metamod shim, cmake), MinHook (vendored C), the existing sigscan infra.

## Global Constraints

- **This stage is read-only + non-blocking.** The detour ALWAYS calls the original trampoline and returns its result. No damage is modified/blocked (that's Stage 3). Any failure path (hook not created, sig not found) degrades to "no hook installed" — the game runs normally.
- **No dlopen cascade.** MinHook is compiled into the shim as static objects (its own `.c` files added to the shim target) — no new shared library. Confirm the shim still `dlopen`s after linking it.
- **Layout-is-data (staged).** Stage 1 may hardcode the `m_flDamage` offset (68, from the catalog) in the shim detour ONLY as a diagnostic, with a `// STAGE-2: resolve via schema` comment. Stage 2 moves the read behind the schema accessor. Do not bake it permanently.
- **The signature is per-update gamedata** — it lives in `gamedata/core.gamedata.jsonc` under `.signatures`, never in code.

### Task 2: Detour `DispatchTraceAttack` + read-only damage log + LIVE proof

```jsonc
    // debug strings). A direct prologue pattern — validated UNIQUE in the pinned libserver.so @0x15ce2e0.

```

If the loader has no `"resolve":"direct"` mode, use whatever the code treats as "the match offset is the target" — read `LoadSignatures`/the resolve switch in `s2script_mm.cpp` and follow it; add a `direct` branch if needed that returns `matchOff` as the address.)

- [ ] **Step 2: Resolve + hook + a read-only detour handler** (`shim/src/s2script_mm.cpp`).

Near the sig-scan block (where `s_pGameEventManager` is resolved), add the resolution for `DispatchTraceAttack` → an absolute address, then MinHook it. The handler ABI: the prologue saves `rdi`(this)→r13, `rsi`→r12, `rdx`→rbx (null-tested) — so at least 3 pointer args. Declare the detour with 4 `void*` params (extra params are harmless and preserve register args), return `int64_t`, always call the trampoline:
```cpp
typedef int64_t (*DispatchTraceAttack_t)(void* thisptr, void* a2, void* a3, void* a4);
static DispatchTraceAttack_t g_origDTA = nullptr;

// STAGE-2 will resolve m_flDamage via schema + dispatch to the core mux. Stage 1 is read-only proof.
static int64_t Detour_DispatchTraceAttack(void* thisptr, void* a2, void* a3, void* a4) {
    // Identify the CTakeDamageInfo among the candidate arg ptrs by reading m_flDamage @68 (catalog offset).
    // Log candidates so the live gate reveals which arg holds the info (a2 vs a3).
    auto rd = [](void* p) -> float { return (p && ((uintptr_t)p > 0x1000)) ? *reinterpret_cast<float*>((char*)p + 68) : -1.f; };
    META_CONPRINTF("[s2script] DTA fired: this=%p a2.dmg=%.1f a3.dmg=%.1f\n", thisptr, rd(a2), rd(a3));
    return g_origDTA(thisptr, a2, a3, a4);   // read-only: always call original
}
```
Install (after `MH_Initialize`, once the address resolves; guard every failure with a log + no-op):
```cpp
    if (dtaAddr) {
        if (MH_CreateHook(reinterpret_cast<void*>(dtaAddr),
                          reinterpret_cast<void*>(&Detour_DispatchTraceAttack),
                          reinterpret_cast<void**>(&g_origDTA)) == MH_OK &&
            MH_EnableHook(reinterpret_cast<void*>(dtaAddr)) == MH_OK) {
            META_CONPRINTF("[s2script] DispatchTraceAttack hooked @%p\n", (void*)dtaAddr);
        } else {
            META_CONPRINTF("[s2script] DispatchTraceAttack hook FAILED (degrade: no damage hook)\n");
        }
    } else {
        META_CONPRINTF("[s2script] DispatchTraceAttack sig unresolved (degrade: no damage hook)\n");
    }
```
