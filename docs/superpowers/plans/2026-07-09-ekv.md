# EKV — CEntityKeyValues-Configured Spawn — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (this project executes via the `Workflow` tool — one agent per task, adversarial review between tasks). Steps use `- [ ]` checkboxes.

**Goal:** `createEntity(className, keyvalues?)` + `EntityRef.spawn(keyvalues?)` — spawn entities configured by a `CEntityKeyValues` built from a `{key: string|number|boolean}` map, so the entity's own `Spawn()` parses them (`DispatchSpawn(entity, keyValues)` instead of `DispatchSpawn(entity, nullptr)`).

**Architecture:** Compile the vendored SDK's own `entity2/entitykeyvalues.cpp` + `tier1/keyvalues3.cpp` into the shim (the CSSharp approach), isolated behind ONE new TU (`shim/src/ekv.cpp` — the only file including `entitykeyvalues.h`), with residual undefined symbols self-shimmed via the `tier1_shims.cpp` pattern and unreachable SDK paths dropped by `-ffunction-sections`/`--gc-sections`. ONE new `S2EngineOps` op (`entity_spawn_kv`, ABI-appended after `entity_fire_input`) carries `(index, serial, count, keys[], types[], values[])` in a single call — the whole `CEntityKeyValues` lifecycle (build → AddRef → `s_pDispatchSpawn(ent, ekv)` → guarded Release) lives inside that one shim call; no handle, no ledger entry, no raw pointer anywhere near JS. **Zero new signatures/offsets** — the op reuses the already-resolved `s_pDispatchSpawn` from the entity-creation slice.

**Tech Stack:** Rust (core, rusty_v8), C++ (shim, hl2sdk), JavaScript (prelude), TypeScript (`.d.ts`).

## Global Constraints

- **Spike-first / stop rule (REVISED — the first spike attempt found the resolution):** Task 1 is the compile-spike. Its exit criterion: the sniper-built shim loads on the live CS2 container (no fatal undefined symbol) AND `[s2script] EKV self-test: OK` appears at Load. Surviving undefined symbols resolve one of THREE ways: (a) a self-containable **leaf** → self-shim in `tier1_shims.cpp` (the ~12 leaves the first spike already identified: `Warning`/`LoggingSystem_Log`/`Plat_*`/`V_StringTo*`/`_V_strncpy`/`V_tier0_memmove`/`LOG_GENERAL`, reuse `s2_MurmurHash2`); (b) **EXPORTED by `libtier0.so` (or `libserver.so`)** → **LIVE-RESOLVE at dlopen — leave it UNDEFINED, do NOT self-shim** (the shim `.so` link already tolerates undefined-resolved-at-load — it's how every engine call already resolves; VERIFIED exported: `CBufferString::Purge` @0x14a810, `CUtlBuffer::CUtlBuffer` @0x273760, `CUtlBuffer::AddNullTermination` — the exact 3 that tripped the first stop rule, plus any `MemAlloc_*`); (c) genuinely absent everywhere (not a leaf, not tier0/server-exported) → **STOP and report** (the fallback = sig-scanning the engine's own EKV internals, a re-design). This is CSSharp's exact approach (it compiles the same `.cpp`s + links tier0). **Heap consistency:** also compile `public/tier0/memoverride.cpp` (CSSharp does) so the shim's `operator new`/`delete` route through the engine allocator — our compiled KV3 code and the engine share ONE heap, closing the cross-heap-free hazard.
- **Boundary (both CI gates must stay green):** the op, natives, and prelude are engine-generic — no CS2 schema class/field names in `core/src` or `packages/entity` (keys are caller parameters). The SDK `.cpp`s compile into the SHIM only. Run `bash scripts/check-core-boundary.sh` and `bash scripts/test-boundary-nameleak.sh`.
- **ABI discipline:** `entity_spawn_kv` is APPENDED after the current last op (`entity_fire_input`, `shim/include/s2script_core.h` — verify with `grep -n "} S2EngineOps" shim/include/s2script_core.h` and confirm `entity_fire_input` is the last field before it) — never reordered — kept identical across the C header, the Rust mirror (`core/src/v8host.rs`), BOTH in-isolate test op-structs (search the two literals containing `entity_fire_input: None,`), and the shim `ops.` assignment.
- **No raw pointer crosses to JS:** the `CEntityKeyValues*` never leaves the single `Shim_EntitySpawnKv` call (it is `void*` even outside `ekv.cpp`); the target entity is `(index, serial)` re-resolved serial-gated per call via the existing `ResolveEntityBySerial`.
- **Degrade-never-crash:** the native is `catch_unwind`-wrapped; unresolved op / stale serial / marshal failure / build failure → `false`/`0`, and a marshal rejection returns `false` BEFORE any engine call (no partial spawn). The queued-spawn edge degrades to a logged bounded leak (never a free-while-held). A failing self-test disables nothing (kv-spawns return `false`; plain `spawn()` unaffected).
- **EKV lifecycle contract (Task 3):** refcount starts 0, `Release()` is `if (--m_nRefCount <= 0) delete this`. Always: `AddRef()` BEFORE `DispatchSpawn` (a balanced engine AddRef/Release can then never engine-delete our-heap memory); AFTER the call, `Release()` only if `!IsQueuedForSpawn()`, else WARN once + deliberately leak.
- **Self-shim vs live-resolve doctrine (Task 1):** self-shim ONLY leaves (`Warning`, `LoggingSystem_Log`, `Plat_FatalErrorFunc`, `V_StringTo*`, `V_strncpy`, `V_tier0_memmove`, `LOG_GENERAL`, `MurmurHash2`) — small, self-contained, no allocator, prototype-matched. Anything EXPORTED by `libtier0.so`/`libserver.so` (the 3 `CBufferString`/`CUtlBuffer` methods, any `MemAlloc_*`) is **LEFT UNDEFINED and live-resolved at dlopen** — self-shimming a real engine method (esp. a string-buffer or allocator) is WRONG (wrong behavior / cross-heap). `memoverride.cpp` (compiled) makes the shim share the engine heap, so this is safe. Treadmill note: live-resolution depends on the export surviving (the 2000870 g_bUpdateStringTokenDatabase lesson) — but `CUtlBuffer`/`CBufferString`/`MemAlloc` are core, universally-used utilities (far more stable than a niche global) and the boot self-test catches a dropped export loudly every update.

- [ ] **Step 3: Write `shim/src/ekv.{h,cpp}`**

`ekv.cpp` includes `entity2/entitykeyvalues.h` and implements:

```cpp
// GameEntitySystem(): the SDK declares this extern and expects the CONSUMER to define it (CSSharp
// defines its own). Bridge to the shim's per-call resolver. Null-safe: the EKV_ALLOCATOR_NORMAL
// paths only ever CHECK it (ctor), never require it — a load-time (pre-map) EKV works fine.
CGameEntitySystem* GameEntitySystem() { return S2_EntitySystemBridge(); }

void* S2EKV_Build(int count, const char* const* keys, const int* types, const char* const* values) {
    if (count < 0 || (count > 0 && (!keys || !types || !values))) return nullptr;
    CEntityKeyValues* kv = new CEntityKeyValues();   // NULL arena + EKV_ALLOCATOR_NORMAL (CSSharp shape)
    for (int i = 0; i < count; ++i) {
        if (!keys[i] || !keys[i][0] || !values[i]) { delete kv; return nullptr; }
        switch (types[i]) {
            case 0: kv->SetString(keys[i], values[i]); break;
            case 1: kv->SetInt(keys[i], (int)strtol(values[i], nullptr, 10)); break;
            case 2: kv->SetFloat(keys[i], strtof(values[i], nullptr)); break;
            case 3: kv->SetBool(keys[i], values[i][0] == '1'); break;
            default: delete kv; return nullptr;
        }
    }
    return kv;
}
void S2EKV_AddRef(void* ekv) { static_cast<CEntityKeyValues*>(ekv)->AddRef(); }
int  S2EKV_ReleaseIfSafe(void* ekv) {
    CEntityKeyValues* kv = static_cast<CEntityKeyValues*>(ekv);
    if (kv->IsQueuedForSpawn()) return 0;   // engine still holds it — caller WARNs once + leaks by design
    kv->Release();                           // our AddRef held it at >=1; this hits 0 -> OUR delete, OUR heap
    return 1;
}
bool S2EKV_SelfTest() {
    CEntityKeyValues kv;                     // stack; dtor runs at scope exit
    kv.SetInt("s2_k1", 42);
    kv.SetString("s2_k2", "ekv");
    kv.SetBool("s2_k3", true);
    kv.SetFloat("s2_k4", 1.5f);
    return kv.GetInt("s2_k1") == 42 && strcmp(kv.GetString("s2_k2"), "ekv") == 0
        && kv.GetBool("s2_k3") && kv.GetFloat("s2_k4") == 1.5f;
}
```

Direct `delete kv` is safe pre-AddRef (refcount 0, never handed out). Note the key hashes go through the shim's own 5D.1 `MurmurHash2LowerCase` (`tier1_shims.cpp`) — byte-for-byte Valve's, so the engine's member lookups match; keys are case-insensitive.

/// Native `__s2_entity_spawn_kv(index, serial, keys[], types[], values[]) -> boolean`. Over the
/// `entity_spawn_kv` op (DispatchSpawn with a shim-built CEntityKeyValues). All three arrays must be
/// same-length; keys/values are strings (interior NUL -> false), types are ints. Degrades to false
/// with no op / stale serial / malformed args (every in-isolate test).
fn s2_entity_spawn_kv(scope: &mut v8::PinScope, args: v8::FunctionCallbackArguments, mut rv: v8::ReturnValue) {

  // EKV marshal: {key: string|number|boolean} -> parallel {keys, types, values} arrays.
  // types: 0=string 1=int 2=float 3=bool; values stringified ("1"/"0" for bool). Inference:
  // integer-in-int32 -> int, other finite number -> float. ANY bad entry (empty key, non-finite,
  // unsupported value type, >256 keys) rejects the WHOLE map (null) — never a partial spawn.
  function __s2_ekv_marshal(kv) {
    var keys = [], types = [], values = [];
    var names = Object.keys(kv);
    if (names.length > 256) return null;
    for (var i = 0; i < names.length; i++) {
      var k = names[i];
      if (!k) return null;
      var v = kv[k], t = typeof v;
      if (t === "string") { types.push(0); values.push(v); }
      else if (t === "number") {
        if (!isFinite(v)) return null;
        if (Number.isInteger(v) && v >= -2147483648 && v <= 2147483647) { types.push(1); values.push(String(v)); }
        else { types.push(2); values.push(String(v)); }
      }
      else if (t === "boolean") { types.push(3); values.push(v ? "1" : "0"); }
      else return null;
      keys.push(k);
    }
    return { keys: keys, types: types, values: values };
  }
  EntityRef.prototype.spawn = function (keyvalues) {
    if (keyvalues === undefined || keyvalues === null) return __s2_entity_spawn(this.index, this.serial);
    if (typeof keyvalues !== "object") return false;
    var m = __s2_ekv_marshal(keyvalues);
    if (m === null) return false;
    if (m.keys.length === 0) return __s2_entity_spawn(this.index, this.serial);
    return __s2_entity_spawn_kv(this.index, this.serial, m.keys, m.types, m.values);
  };
  function createEntity(className, keyvalues) {
    var ref = __s2_entity_create(String(className));
    if (!ref) return null;
    if (keyvalues !== undefined && keyvalues !== null) {
      // With kv: non-null result = a LIVE, SPAWNED entity. On spawn failure, remove the unspawned
      // entity (hygiene — never strand a half-configured entity) and return null.
      if (!ref.spawn(keyvalues)) { ref.remove(); return null; }
    }
    return ref;
  }

// spawn-with-keyvalues: DispatchSpawn(ent, CEntityKeyValues) — the entity's own Spawn() parses the
// keys (the SM DispatchKeyValue / CSSharp DispatchSpawn(kv) mechanism). The EKV's whole lifecycle is
// inside this call: build -> AddRef (so a balanced engine AddRef/Release can never delete our-heap
// memory mid-call) -> DispatchSpawn -> Release IF the engine isn't still holding it queued (then we
// WARN once + deliberately leak one small object — a bounded leak beats a UAF/cross-heap free).
static int Shim_EntitySpawnKv(int index, int serial, int count,
                              const char* const* keys, const int* types, const char* const* values) {
    if (!s_pDispatchSpawn) return 0;
    CEntityInstance* ent = ResolveEntityBySerial(index, serial);
    if (!ent) return 0;
    void* ekv = S2EKV_Build(count, keys, types, values);
    if (!ekv) return 0;
    S2EKV_AddRef(ekv);
    s_pDispatchSpawn(ent, ekv);
    if (!S2EKV_ReleaseIfSafe(ekv)) {
        static bool s_warnedEkvLeak = false;
        if (!s_warnedEkvLeak) {
            s_warnedEkvLeak = true;
            META_CONPRINTF("[s2script] WARN: engine kept a spawn CEntityKeyValues queued — leaking it (once-per-boot notice)\n");
        }
    }
    return 1;
}
