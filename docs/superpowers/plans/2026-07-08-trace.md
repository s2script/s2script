# Ray tracing (`@s2script/trace`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** `@s2script/trace` (`Trace.line`/`ray`/`hull` → `TraceHit`) + `pawn.aimTrace()`, over one engine-generic `trace_shape` op that calls CS2's `CNavPhysicsInterface::TraceShape`, resolved by an RTTI vtable-by-name scan.

**Architecture:** A new shim RTTI vtable-resolution capability + a `trace_shape` `S2EngineOps` op (shim impl) + a core `__s2_trace` native + the `@s2script/trace` prelude module + a `pawn.aimTrace` CS2 helper. Reference mechanism: FUNPLAY-pro-CS2/Ray-Trace (vendors DynLibUtils).

**Tech Stack:** C++ (shim, hl2sdk), Rust+V8 (core), TypeScript (`.d.ts` + prelude).

## Global Constraints

- **Boundary:** the trace op + `@s2script/trace` are ENGINE-GENERIC (Source 2 physics) — core/shim only, NO CS2 names. Only `pawn.aimTrace` (eye offset / `eyeAngles` field names) is in the CS2 layer (`games/cs2/js/pawn.js` + `packages/cs2/index.d.ts`). Both `scripts/check-core-boundary.sh` and `test/…nameleak` MUST stay green.
- **No raw pointer crosses to JS.** The op returns a flat `S2TraceResult`; the hit entity crosses as an `(index, serial)` `EntityRef` (decode the `CEntityInstance*`'s `GetRefEHandle().ToInt()` in the shim → core builds a serial-gated `EntityRef`, the `DamageInfo.victim` pattern).
- **Degrade-never-crash.** Unresolved vtable/typeinfo → the op is disabled → the native returns a MISS `TraceHit` (`didHit:false, fraction:1, endPos:end, normal:(0,0,0), entity:null, allSolid:false`). Guarded by `!s_pTraceShape`.
- **Self-resolve doctrine.** The vtable is RTTI-resolved in OUR `libserver.so`; the vtable INDEX (gamedata, `linuxsteamrt64:5`) is validated at load (`.text`-membership) + a first-use smoke test.
- **ABI append order:** the `trace_shape` fn-ptr is appended AFTER `config_write_file` (the current last `S2EngineOps` field) in the SAME position across the C header, the Rust mirror, and BOTH in-isolate test op-structs.

- **One sniper build** (shim + core .so) after all tasks — a bullseye build (`libs2script_core.so` GLIBC ≤2.30, `s2script.so` ≤2.14). Deploy + live gate are the controller's job, NOT a task.

## Reference facts (recon-confirmed)

- Resolution: RTTI scan. In the pinned `libserver.so` the `.symtab` is stripped and game vtables are NOT in `.dynsym` (`dlsym` fails), but the typeinfo-name string **`20CNavPhysicsInterface`** IS in `.rodata`. Algorithm (port DynLibUtils `vendor/dynlibutils/module_linux.cpp` — fetchable at `https://raw.githubusercontent.com/FUNPLAY-pro-CS2/Ray-Trace/main/vendor/dynlibutils/module_linux.cpp`; and `module.h`/`memaddr.h` in the same dir):
  1. Get the module's `.rodata` and `.data.rel.ro` (+ `.data.rel.ro.local`) section ranges (parse the ELF section headers of the mapped module; the shim already walks segments via `dl_iterate_phdr` for `FindModuleText` — extend to sections, or mmap+parse the on-disk `.so` at the module's base like DynLibUtils does).
  2. Decorate the name: `<len><name>` = `"20CNavPhysicsInterface"`. Find that byte string in `.rodata` → `typeInfoName` address.
  3. In `.data.rel.ro`, find an 8-byte pointer whose value == `typeInfoName`. That location − `0x8` = the `type_info` object.
  4. In `.data.rel.ro`(+`.local`), find an 8-byte pointer whose value == the `type_info` object AND the qword at (that location − `0x8`) == 0 (offset-to-top of the primary base). That location + `0x8` = the **vtable's first virtual function** (`vtable`). 
  5. `TraceShape = vtable[gamedata index]` (Linux 5).
- Call: `bool TraceShape(void* /*this=nullptr*/, Ray_t& ray, Vector& start, Vector& end, CTraceFilter* filter, CGameTrace* trace)`.
- Filter: a `CTraceFilterEx : CTraceFilter` with `m_nInteractsAs=0`, `m_nInteractsWith=<mask>`, `m_nInteractsExclude=<exclude>`, and an optional ignore `CBaseEntity*`. Masks are `InteractionLayers` uint64 bitsets; predefined set (the module holds the 7 constants).
- Structs `Ray_t` / `CTraceFilter` / `CGameTrace` are Source-2-SDK types — source them from the reference's includes / the vendored hl2sdk; if absent, define minimal ABI-correct versions (see Task 1 Step 3). Ray_t line trace = `Init(mins={0,0,0}, maxs={0,0,0})`.

### Task 1: Shim — RTTI vtable resolution + trace structs + the `trace_shape` op

**Files:** Create `shim/src/vtable.cpp` + `shim/src/vtable.h` (the RTTI helper) and `shim/src/trace.cpp` + `shim/src/trace.h` (the structs + the trace call); Modify `shim/include/s2script_core.h` (the `S2TraceResult` struct + the `trace_shape` fn-ptr appended to `S2EngineOps`), `shim/src/s2script_mm.cpp` (resolve the vtable at Load, wire the op, the `.text` validation + the gamedata-validation line), `gamedata/core.gamedata.jsonc` (the `CNavPhysicsInterface_TraceShape` offset + the `CNavPhysicsInterface` interface-name note), `shim/CMakeLists`-equivalent build list (add the new .cpp).

**Interfaces:**
- Produces (for Task 2): the C ABI — in `s2script_core.h`:
  ```c
  typedef struct { int didHit; float fraction; float endpos[3]; float normal[3];
                   int allSolid; int hitEntHandle; } S2TraceResult;
  // appended to S2EngineOps AFTER config_write_file:
  int (*trace_shape)(const float* start, const float* end, const float* mins, const float* maxs,
                     unsigned long long interactsWith, unsigned long long interactsExclude,
                     int ignoreEntIdx, int ignoreEntSerial, S2TraceResult* out);
  ```
  Contract: returns 1 and fills `*out` on a completed trace (didHit=trace hit, fraction/endpos/normal/allSolid from `CGameTrace`, `hitEntHandle` = the hit `CEntityInstance`'s `GetRefEHandle().ToInt()` or `-1`); returns 0 (op unavailable / vtable unresolved) leaving `*out` untouched.

- [ ] **Step 1: Port the RTTI vtable-by-name helper** into `shim/src/vtable.{h,cpp}`: `void** GetVTableByName(const char* module, const char* className)` implementing the 5-step algorithm above (fetch DynLibUtils `module_linux.cpp`/`module.h`/`memaddr.h` and adapt to the shim's style; reuse the existing largest-`PF_X`-module selection from `FindModuleText` for `libserver.so` — the Metamod proxy pitfall). Return `nullptr` if not found. Add a tiny host-`g++` test (compile-only or a stub) if practical; otherwise validated live.

- [ ] **Step 2: Resolve at Load.** In `s2script_mm.cpp` Load, after the schema/other resolves: `void** vt = GetVTableByName("libserver.so", "CNavPhysicsInterface");` then `s_pTraceShape = vt ? (TraceShapeFn)vt[gameconfig offset "CNavPhysicsInterface_TraceShape"] : nullptr;`. Validate `s_pTraceShape` lands in `libserver.so`'s `PF_X` range (reuse `FindModuleText`'s bounds); if not, null it. Emit a `=== GAMEDATA VALIDATION ===`-style line (`trace: OK (RTTI CNavPhysicsInterface idx N)` / `trace: MISSING`) so it joins the boot validation count.

- [ ] **Step 3: Define the trace structs** in `shim/src/trace.{h,cpp}` — `Ray_t`, `CTraceFilter`, `CTraceFilterEx : CTraceFilter` (`m_nInteractsAs`/`m_nInteractsWith`/`m_nInteractsExclude` + the ignore entity), `CGameTrace` — sourced from the reference's includes / hl2sdk; if the vendored hl2sdk lacks them, define minimal ABI-correct layouts (Ray_t: the Source-2 ray with `Init(mins,maxs)`; CGameTrace: enough to read endpos/fraction/normal/hitentity/allsolid at their offsets). Include the 7 predefined `InteractionLayers` masks as `constexpr uint64_t`. **Note in a comment that the exact `CGameTrace` field offsets are live-validated (the smoke test / the live gate) — a per-update change is a struct regeneration.**

- [ ] **Step 4: Implement the `trace_shape` op** (`shim/src/trace.cpp`, wired in `s2script_mm.cpp`'s `S2EngineOps` init): guard `!s_pTraceShape` → return 0. Build `Ray_t ray; ray.Init(Vector(mins), Vector(maxs));` a `CTraceFilterEx` (set the 3 mask fields; resolve the ignore entity from `(ignoreEntIdx, ignoreEntSerial)` via the existing serial-gated entity lookup → a `CBaseEntity*` or null); a `CGameTrace trace;`. Call `s_pTraceShape(nullptr, ray, Vector(start), Vector(end), &filter, &trace);`. Copy into `*out`: `didHit` (fraction<1 || allsolid || the trace's DidHit), `fraction`, `endpos[3]`, `normal[3]`, `allSolid`, `hitEntHandle` (the hit entity's `GetRefEHandle().ToInt()`, or -1). Return 1.

- [ ] **Step 5: Gamedata + build.** Add to `core.gamedata.jsonc` `offsets`: `"CNavPhysicsInterface_TraceShape": { "linuxsteamrt64": 5 }` with a comment (borrowed index, validated at load + live). Add the new .cpp files to the shim build. Commit `feat(trace): shim RTTI vtable resolution + CNavPhysicsInterface::TraceShape op`.

### Task 2: Core — the `trace_shape` ABI mirror, `__s2_trace` native, `@s2script/trace` module, `QAngle`→forward

**Files:** Modify `core/src/v8host.rs` (the Rust `S2EngineOps` mirror + BOTH in-isolate test op-structs; the `s2_trace` native; the `@s2script/trace` prelude module; register the module in the `s2require` map; add `QAngle.forward`/an angle→vector helper to the `@s2script/math` prelude), `packages/trace/{package.json,index.d.ts}` (new, types-only), `packages/math/index.d.ts` (the forward helper). Modify the CLI externalize list only if `@s2script/trace` isn't wildcard-covered (it is — `@s2script/*`).

**Interfaces:**
- Consumes (from Task 1): the `S2TraceResult` layout + the `trace_shape` op contract.
- Produces: `globalThis.__s2pkg_trace = { Trace, TraceMask }`; `Trace.line/ray/hull(...) → TraceHit`; `TraceHit = { didHit, fraction, endPos: Vector, normal: Vector, entity: EntityRef|null, allSolid }`.

- [ ] **Step 1: ABI mirror.** Add the `trace_shape` fn-ptr to the Rust `S2EngineOps` struct (mirror the C signature; `S2TraceResult` as a `#[repr(C)]` Rust struct or an out-pointer to a stack buffer the native reads) AFTER `config_write_file`, and add `trace_shape: None` to BOTH test op-structs. `cargo test` must still compile.

- [ ] **Step 2: `s2_trace` native** (`set_native "__s2_trace"`): args `(startArr, endArr, minsArr, maxsArr, interactsWith:Number, interactsExclude:Number, ignoreIdx:Number, ignoreSerial:Number)`. `catch_unwind`. If no `trace_shape` op → build a MISS `TraceHit` (didHit false, fraction 1, endPos from `endArr`, normal (0,0,0), entity null, allSolid false). Else call the op with a stack `S2TraceResult`, then build a `TraceHit` v8 object: `endPos`/`normal` via `__s2pkg_math`'s `Vector`, `entity` = `hitEntHandle < 0 ? null : __s2_handle_decode(hitEntHandle)` (→ a serial-gated `EntityRef` | null), `didHit`/`allSolid` bool, `fraction` number.

- [ ] **Step 3: `@s2script/trace` prelude module.** In the core prelude JS, define `__s2pkg_trace`:
  - `TraceMask` = an object of the 7 named uint64 masks as JS numbers (values from Task 1's `constexpr` set — copy them; assert < 2^53).
  - `Trace.line(start, end, opts)` → `__s2_trace([start.x,start.y,start.z],[end.x,end.y,end.z],[0,0,0],[0,0,0], mask(opts), exclude(opts), ig(opts).idx, ig(opts).serial)` where `mask(opts)=opts?.mask ?? TraceMask.ShotPhysics`, `ig` returns the ignoreEntity's `(index,serial)` or `(-1,-1)`.
  - `Trace.ray(start, angles, distance, opts)` → `end = { x: start.x+f.x*distance, y: start.y+f.y*distance, z: start.z+f.z*distance }` where `f = __s2pkg_math.forwardVector(angles)`; then `Trace.line(start, end, opts)`.
  - `Trace.hull(start, end, mins, maxs, opts)` → the op with the given mins/maxs.
  - Register `trace` in the `s2require`/`__s2pkg_*` map (engine-generic, like `math`/`entity`).

- [ ] **Step 4: `@s2script/math` forward helper.** Add `forwardVector(a: QAngle): Vector` to the math prelude + `.d.ts`: `p=a.pitch*PI/180, y=a.yaw*PI/180; return new Vector(cos(p)*cos(y), cos(p)*sin(y), -sin(p))`.

- [ ] **Step 5: Types + tests.** `packages/trace/package.json` (types-only) + `index.d.ts` (`Trace`, `TraceHit`, `TraceMask`, `TraceOptions`). Add in-isolate `cargo test`s: `__s2_trace` with no op → the MISS shape (didHit false, fraction 1, entity null); the `TraceHit` object has Vector endPos/normal; `forwardVector` for a known angle (e.g. yaw 0 pitch 0 → (1,0,0); yaw 90 → ~(0,1,0)); the `TraceMask` values. Commit `feat(trace): @s2script/trace module + __s2_trace native + QAngle forward + tests`.
